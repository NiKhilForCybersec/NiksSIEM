Q: If today is Wednesday 2PM, what does earliest=-1d@d latest=@d return?
  • A) Tuesday 2PM to Wednesday 2PM
  • B) Tuesday midnight to Wednesday midnight
  • C) Monday midnight to Tuesday midnight
  • D) Wednesday midnight to now
✓ B) -1d@d = go back 1 day, snap to midnight = Tuesday 00:00. @d = snap to today's midnight = Wednesday 00:00.

3.2 Essential Field Commands

CommandSyntaxPurpose
fields| fields user, status, bytesINCLUDE only these fields (improves performance)

table| table user, status, bytesFormat output as table (cosmetic)

sort| sort -bytesSort descending (- prefix)

fields vs table:
  • fields - Reduces data EARLY = faster performance
  • table - Formats output at END = no performance benefit
  • Best practice: Use fields early, table last

Q: To keep only the MOST RECENT event per user, which command is correct?
  • A) | dedup user
  • B) | dedup user sortby -_time
  • C) | sort -_time | head 1
  • D) | stats latest(user)
✓ B) dedup user sortby -_time sorts by time descending first, then keeps first (most recent) per user.
