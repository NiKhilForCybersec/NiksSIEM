Splunk Cloud Architecture

üîë Key Cloud Concepts
  • IDM (Inputs Data Manager): Receives data from forwarders
  • SmartStore: S3-based storage for indexed data
  • ACS (Admin Config Service): API for cloud administration
  • Victoria Experience: Cloud-native with enhanced features
Q: In Splunk Cloud, who is responsible for infrastructure upgrades?
  • A) Splunk manages all infrastructure and upgrades
  • B) Customer schedules and performs upgrades
  • C) Shared responsibility for upgrades
  • D) Third-party vendor manages upgrades
‚úì A) Splunk Cloud is fully managed - Splunk handles ALL infrastructure, upgrades, patching, and maintenance. Customers focus on data onboarding and content creation.

üîë SmartStore

Splunk Cloud uses SmartStore - data is stored in S3 with local caching on indexers. This provides:

  • Virtually unlimited storage capacity
  • Cost-effective long-term retention
  • Automatic tiering (hot cache ‚Üí warm S3)
Sobeys Cloud Indexes
  • pos_transactions - POS data (90-day retention)
  • security_events - Security logs (1-year retention)
  • network_traffic - Firewall/proxy (30-day retention)

Domain 6: Forwarders to Cloud

15% (~9 questions)

‚ö†Ô∏è Highest Priority Domain (Tied)!

Master forwarder configuration for sending data to Splunk Cloud.

Connecting Forwarders to Cloud

Port 9997
Standard Forwarding

Universal Forwarders connect to cloud IDM on port 9997 (encrypted by default)

TLS Required
Encryption

All connections to Splunk Cloud MUST use TLS. No unencrypted connections allowed.

outputs.conf for Cloud

Critical Config
Universal Forwarder ‚Üí Splunk Cloud
[tcpout] defaultGroup = splunkcloud [tcpout:splunkcloud] server = inputs1.<stack>.splunkcloud.com:9997, inputs2.<stack>.splunkcloud.com:9997 # SSL is REQUIRED for Splunk Cloud sslCertPath = $SPLUNK_HOME/etc/apps/splunkcloud/default/cacert.pem sslRootCAPath = $SPLUNK_HOME/etc/apps/splunkcloud/default/cacert.pem sslVerifyServerCert = true useACK = true
üîë Universal Forwarder Credentials Package

Splunk provides a credentials package (splunkclouduf.spl) that contains:

  • Pre-configured outputs.conf
  • SSL certificates
  • Server addresses

Download: Splunk Cloud UI ‚Üí Universal Forwarder ‚Üí Download Credentials

Q: What is required for forwarder connections to Splunk Cloud?
  • A) Unencrypted TCP on port 514
  • B) TLS-encrypted connection on port 9997
  • C) VPN tunnel only
  • D) HTTP on port 80
‚úì B) Splunk Cloud requires TLS-encrypted connections. Forwarders connect to IDM on port 9997 with SSL enabled. The credentials package provides the necessary certificates.

Cloud Admin Exam Tips

  • Focus on high-weight: Forwarders (15%) + Data Input (15%) + Architecture (13%) = 43%
  • Know the responsibility model: Splunk manages infrastructure, customer manages data & content
  • sc_admin role: Required for ACS API access (cloud-specific)
  • TLS required: All connections to cloud must be encrypted
  • Credentials package: How to configure forwarders for cloud
  • HEC: Port 443, HTTPS only, for application data
  • Support process: Know what requires a ticket vs self-service
