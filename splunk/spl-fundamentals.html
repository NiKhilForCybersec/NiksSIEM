SPL Fundamentals

Search Processing Language (SPL) is the query language for Splunk. Master the fundamentals to effectively search, filter, and analyze security data.

SPL Query Structure

SPL Pipeline Flow
Search
index=main
|
Filter
where, search
|
Transform
eval, rex
|
Aggregate
stats, timechart
|
Format
table, sort

Key Concept: The Pipe

SPL uses the pipe character | to pass results from one command to the next. Each command transforms the data and passes it downstream.

index=main sourcetype=syslog | where severity="error" | stats count by host | sort -count

Basic Search Commands

Index and Sourcetype Selection

Basic Search Patterns
// Search specific index
index=security

// Search specific sourcetype
index=security sourcetype=WinEventLog:Security

// Search multiple indexes
index=security OR index=network

// Wildcard search
index=sec* sourcetype=*syslog*

// Time range (relative)
index=security earliest=-24h latest=now

// Time range (absolute)
index=security earliest="01/15/2024:00:00:00" latest="01/15/2024:23:59:59"

Field Filtering

Filtering Methods
// Field equals value
index=security EventCode=4625

// Field contains string
index=security "failed login"

// Field NOT equals
index=security EventCode!=4624
// OR
index=security NOT EventCode=4624

// Multiple values (OR)
index=security EventCode IN (4624, 4625, 4634)

// Wildcard matching
index=security user=admin*

// Using where command (more powerful)
index=security 
| where EventCode==4625 AND user!="SYSTEM"
| where cidrmatch("10.0.0.0/8", src_ip)

Essential SPL Commands

Filtering Commands
  • search - Filter by keyword
  • where - Filter by expression
  • dedup - Remove duplicates
  • head - First N results
  • tail - Last N results
Aggregation Commands
  • stats - Calculate statistics
  • count - Count events
  • timechart - Time-based chart
  • chart - Create charts
  • top - Top N values
Transform Commands
  • eval - Create/modify fields
  • rex - Regex extraction
  • rename - Rename fields
  • replace - Replace values
  • fillnull - Fill null values
Output Commands
  • table - Display as table
  • fields - Select fields
  • sort - Sort results
  • outputlookup - Save to lookup
  • format - Format output

The stats Command

stats Examples
// Count events by field
index=security EventCode=4625
| stats count by src_ip

// Multiple aggregations
index=security EventCode=4625
| stats count as FailedLogins, 
        dc(user) as UniqueUsers,
        values(user) as TargetedUsers,
        earliest(_time) as FirstAttempt,
        latest(_time) as LastAttempt
  by src_ip

// Common stats functions:
// count - count of events
// dc(field) - distinct count
// sum(field) - sum of values
// avg(field) - average
// min(field), max(field) - min/max
// values(field) - list of values
// list(field) - list with duplicates
// first(field), last(field) - first/last value

The eval Command

eval Examples
// Create new field
index=network
| eval total_bytes = bytes_in + bytes_out

// Conditional logic (if/else)
index=security EventCode=4625
| eval severity = if(count > 10, "High", "Low")

// Case statement
index=security
| eval event_type = case(
    EventCode==4624, "Login Success",
    EventCode==4625, "Login Failure",
    EventCode==4634, "Logoff",
    true(), "Other"
)

// String functions
| eval domain = mvindex(split(user, "\\"), 0)
| eval username = mvindex(split(user, "\\"), 1)
| eval lower_user = lower(username)

// Time functions
| eval hour = strftime(_time, "%H")
| eval day_of_week = strftime(_time, "%A")
| eval is_weekend = if(match(day_of_week, "Saturday|Sunday"), "Yes", "No")

Security Use Case Examples

Example 1: Failed Login Analysis

Brute Force Detection
index=wineventlog sourcetype=WinEventLog:Security EventCode=4625
| stats count as FailedAttempts,
        dc(Account_Name) as UniqueAccounts,
        values(Account_Name) as TargetAccounts,
        earliest(_time) as FirstAttempt,
        latest(_time) as LastAttempt
  by Source_Network_Address
| where FailedAttempts > 10
| eval AttackDuration = LastAttempt - FirstAttempt
| eval AttackDuration_Minutes = round(AttackDuration/60, 2)
| sort - FailedAttempts
| table Source_Network_Address, FailedAttempts, UniqueAccounts, 
       TargetAccounts, AttackDuration_Minutes

Example 2: Suspicious Process Execution

PowerShell Hunting
index=wineventlog sourcetype=WinEventLog:Security EventCode=4688
| search New_Process_Name="*powershell.exe"
| rex field=Process_Command_Line "(?i)(-enc|-encoded|-e\s)"
| where isnotnull(_rex)
| eval suspicious_flags = if(
    match(Process_Command_Line, "(?i)(downloadstring|invoke-expression|iex|bypass)"),
    "HIGH", "MEDIUM"
)
| table _time, ComputerName, Account_Name, New_Process_Name, 
       Process_Command_Line, suspicious_flags
| sort - _time

Example 3: Network Traffic Analysis

Top Talkers
index=network sourcetype=pan:traffic
| stats sum(bytes_in) as TotalBytesIn,
        sum(bytes_out) as TotalBytesOut,
        count as Sessions,
        dc(dest_ip) as UniqueDestinations
  by src_ip
| eval TotalBytes = TotalBytesIn + TotalBytesOut
| eval TotalGB = round(TotalBytes/1024/1024/1024, 2)
| sort - TotalBytes
| head 20
| table src_ip, Sessions, UniqueDestinations, TotalGB

Interview Questions

search command:

  • Works on raw events (before field extraction)
  • Supports wildcards and boolean operators
  • Case-insensitive by default
  • Better for keyword searches

where command:

  • Works on extracted fields
  • Supports eval expressions
  • Case-sensitive (use lower() for insensitive)
  • Better for field comparisons, calculations
  • Supports functions like cidrmatch(), isnotnull()

Optimization techniques:

  • Specify index and sourcetype: Always include in base search
  • Limit time range: Shorter = faster
  • Filter early: Put restrictive filters before transforming commands
  • Use fields command: Limit fields passed through pipeline
  • Avoid wildcards: Leading wildcards (*keyword) are slow
  • Use tstats: For indexed fields, much faster than search
  • Accelerate data models: Pre-summarize for fast queries
