SPL Fundamentals

SPL (Search Processing Language) is Splunk's query language. Master these fundamentals to effectively search, filter, transform, and visualize security data.

Basic Query Structure

SPL Query Anatomy
index=security sourcetype=WinEventLog:Security    // 1. Search criteria
| where EventCode=4625                            // 2. Filter results
| stats count by Account_Name, src_ip             // 3. Aggregate
| sort -count                                     // 4. Sort descending
| head 10                                         // 5. Top 10

Essential Commands

Security Search Examples

Common Security Queries
// Failed logins (Windows)
index=security sourcetype=WinEventLog:Security EventCode=4625 earliest=-24h
| stats count by Account_Name, Source_Network_Address
| where count > 10
| sort -count

// Successful login after failures (brute force success)
index=security sourcetype=WinEventLog:Security (EventCode=4625 OR EventCode=4624) earliest=-1h
| stats count(eval(EventCode=4625)) as failures, count(eval(EventCode=4624)) as successes by Account_Name
| where failures > 5 AND successes > 0

// PowerShell execution
index=security sourcetype=WinEventLog:Security EventCode=4688
| search Process_Name="*powershell*"
| table _time, Computer, Account_Name, Process_Command_Line

// Outbound connections to rare destinations
index=network sourcetype=firewall action=allowed direction=outbound earliest=-24h
| stats count by dest_ip
| where count < 5
| lookup geoip ip AS dest_ip OUTPUT country
| table dest_ip, count, country

// User activity timeline
index=* user=admin earliest=-24h
| table _time, index, sourcetype, action, src_ip, dest
| sort _time

Interview Questions

search: Filters raw events early in pipeline, uses indexed fields, faster for initial filtering

where: Filters after field extraction, uses eval expressions, more flexible but runs later

Best practice: Use search for initial filtering (leverages index), where for complex conditions after field extraction

rex command: Extract using regex with named capture groups

| rex "user=(?<username>[^\s]+)"
| rex field=_raw "src=(?<src_ip>\d+\.\d+\.\d+\.\d+)"

Permanent extraction: Use transforms.conf with REGEX and FORMAT

index=auth sourcetype=vpn action=success earliest=-24h
| iplocation src_ip
| stats dc(Country) as country_count, values(Country) as countries by user
| where country_count > 1
| table user, country_count, countries
