Security Dashboards

Build effective security dashboards in Splunk for monitoring, reporting, and investigation. Use SimpleXML or Dashboard Studio for rich visualizations.

Dashboard Types

TypeTechnologyBest For
SimpleXMLXML-based, classicQuick dashboards, ES integration
Dashboard StudioJSON-based, modernRich visualizations, interactivity
HTML DashboardsCustom HTML/JSHighly customized UIs

SimpleXML Example

Security Overview Dashboard
<dashboard>
  <label>Security Overview</label>
  <row>
    <panel>
      <title>Failed Logins (24h)</title>
      <single>
        <search>
          <query>index=security EventCode=4625 | stats count</query>
          <earliest>-24h</earliest>
        </search>
        <option name="colorMode">block</option>
        <option name="rangeColors">["0x65a637","0xf58f39","0xd93f3c"]</option>
        <option name="rangeValues">[100,500]</option>
      </single>
    </panel>
    <panel>
      <title>Open Notables</title>
      <single>
        <search>
          <query>| `notable` | search status="new" OR status="in progress" | stats count</query>
        </search>
      </single>
    </panel>
  </row>
  <row>
    <panel>
      <title>Top Failed Users</title>
      <chart>
        <search>
          <query>index=security EventCode=4625 | top limit=10 Account_Name</query>
          <earliest>-24h</earliest>
        </search>
        <option name="charting.chart">bar</option>
      </chart>
    </panel>
  </row>
</dashboard>

Dashboard Best Practices

Purpose-Driven

Each dashboard should answer specific questions. Don't create everything dashboards.

Performance

Use base searches shared across panels. Avoid real-time for non-critical views.

Drilldowns

Enable clicks to investigate. Link panels to searches or other dashboards.

Tokens

Use input tokens for dynamic filtering. Time picker, dropdown filters.

Interview Questions

  • Use base searches: One search feeds multiple panels
  • Accelerate data models: Use tstats instead of raw searches
  • Narrow time range: Default to shorter windows
  • Use report acceleration: Pre-compute expensive searches
  • Avoid real-time: Use scheduled refresh instead
  • Limit results: Use | head or | top to limit data
