Key Concepts

ComponentDescription
EventsIngested alerts from SIEM, email, etc.
ContainersCase/incident containing related events
ArtifactsIOCs extracted from events (IPs, hashes, domains)
PlaybooksAutomated workflows (visual or Python)
AppsIntegrations with security tools
ActionsAPI calls to external systems

Example Playbook: Phishing Triage

  1. Trigger: Phishing alert from email gateway
  2. Extract artifacts: URLs, attachments, sender
  3. Enrich: Check URLs in VirusTotal, URLhaus
  4. Detonate: Submit attachment to sandbox
  5. Decision: If malicious → Contain, else → Close
  6. Contain: Block sender, delete emails, notify user
  7. Document: Create Jira ticket, update SIEM

Interview Preparation

How does Splunk SOAR integrate with Splunk ES?

SOAR and ES integrate to automate incident response:

ES → SOAR: Notable events in ES can trigger SOAR playbooks via the "Send to SOAR" adaptive response action. The notable becomes a container in SOAR with artifacts (IPs, users, hashes) extracted automatically.

SOAR → ES: SOAR can update notable status, add comments, and close incidents in ES. Playbook results flow back to the notable for analyst visibility.

Automation examples:

  • Auto-enrich all notables with TI lookups
  • Phishing alerts → extract IOCs → block → close if confirmed malicious
  • High-severity alerts → auto-create ServiceNow ticket
  • Malware detection → isolate endpoint via CrowdStrike
