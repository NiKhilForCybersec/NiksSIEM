Splunk SOAR - Complete Automation Guide

Comprehensive guide to Splunk SOAR (formerly Phantom). Covers playbook creation, app integration, automation workflows, and Enterprise Security integration.

What is Splunk SOAR?

SOAR = Security Orchestration, Automation, and Response

  • Orchestration: Connect and coordinate actions across 300+ security tools
  • Automation: Playbooks automate repetitive SOC tasks
  • Response: Execute containment, enrichment, and remediation actions
  • Case Management: Track investigations, assign tasks, document findings

Playbook Development

Playbooks → + New Playbook

Playbook Structure

  • Trigger: What starts the playbook (event, manual, schedule, API)
  • Actions: Steps that execute against apps/assets
  • Decisions: Conditional logic (if/then)
  • Filters: Process only matching artifacts
  • Code Blocks: Custom Python for complex logic
  • Prompts: Request analyst input

Example Playbook: Phishing Investigation

Phishing_Triage (Auto-runs on phishing notable)

TRIGGER: Event type = "phishing" BLOCK 1: Extract Artifacts ├── Get sender email, URLs, attachments from event BLOCK 2: Reputation Checks (Parallel) ├── ACTION: VirusTotal → url_reputation (all URLs) ├── ACTION: VirusTotal → file_reputation (attachment hashes) ├── ACTION: AbuseIPDB → lookup_ip (sender IP) BLOCK 3: Decision - Is Malicious? ├── IF any_reputation_score > 5: │ ├── BLOCK 4: Containment │ │ ├── ACTION: Email Gateway → block_sender │ │ ├── ACTION: Proxy → block_urls │ │ └── ACTION: Email → delete_from_all_mailboxes │ └── BLOCK 5: Notify │ └── ACTION: SMTP → send_email to SOC └── ELSE: └── BLOCK 6: Escalate for Review └── ACTION: Create prompt for analyst

Example Playbook: Endpoint Containment

Endpoint_Auto_Contain

TRIGGER: Event type = "malware" AND severity = "critical" BLOCK 1: Get Host Details ├── ACTION: Splunk → run_query │ Query: index=endpoint host={artifact.hostname} | head 1 BLOCK 2: Validate Not Critical System ├── CODE BLOCK: Check hostname not in critical_systems list │ critical_systems = ["DC01", "EXCHANGE", "SQL-PROD"] │ if hostname in critical_systems: │ return "SKIP" BLOCK 3: Contain Endpoint ├── ACTION: CrowdStrike → contain_device │ Parameters: device_id = {action_result.device_id} BLOCK 4: Create Ticket ├── ACTION: ServiceNow → create_ticket │ Short Description: "Endpoint Contained - {hostname}" │ Description: "Auto-contained due to critical malware detection" │ Priority: 1 BLOCK 5: Notify SOC ├── ACTION: Slack → send_message │ Channel: #soc-alerts │ Message: "⚠️ {hostname} auto-contained. Ticket: {ticket_id}"

Interview Questions

SOAR (Security Orchestration, Automation, and Response) automates SOC workflows:

  • Orchestration: Coordinates actions across 300+ tools via apps
  • Automation: Playbooks auto-execute enrichment, triage, response
  • Response: Takes containment actions (isolate host, block IP, disable user)

ES Integration:

  1. Configure Splunk app in SOAR with ES credentials
  2. Create data source to ingest notable events
  3. Playbooks trigger on notable event type
  4. SOAR can update notable status, add comments, change urgency
  1. Trigger: Event label = "phishing" (from email gateway or ES notable)
  2. Extract artifacts: Parse sender, URLs, attachments from event
  3. Parallel enrichment: VirusTotal for URLs/files, AbuseIPDB for sender IP
  4. Decision block: If malicious scores exceed threshold...
  5. Containment: Block sender, quarantine email, block URLs at proxy
  6. Notification: Create ticket, alert SOC, send report to user
  7. Close loop: Update ES notable with findings and status

Key apps: VirusTotal, Email Gateway, Proxy, ServiceNow, SMTP

Extended Best Practices

Operational Excellence

  • Documentation Standards: Maintain comprehensive runbooks for all procedures. Include step-by-step instructions, expected outcomes, and rollback procedures.
  • Change Management: Test all changes in non-production environments first. Document changes and obtain approvals before production deployment.
  • Monitoring Strategy: Implement proactive monitoring with clear thresholds. Set up alerting for critical metrics before they become problems.
  • Regular Reviews: Conduct periodic audits of configurations, performance metrics, and security posture. Schedule quarterly reviews at minimum.
  • Continuous Training: Keep team skills current with platform updates. Allocate time for learning new features and best practices.
  • Knowledge Sharing: Document lessons learned and share across the team. Build institutional knowledge through wikis and team sessions.

Security Hardening

  • Principle of Least Privilege: Grant minimum necessary permissions for each role. Review and revoke unnecessary access regularly.
  • Comprehensive Audit Logging: Enable detailed audit trails for all administrative actions. Retain logs according to compliance requirements.
  • Data Protection: Encrypt sensitive data in transit and at rest. Implement data classification and handling policies.
  • Access Reviews: Conduct regular reviews of user access and permissions. Implement automated access certification workflows.
  • Secrets Management: Use secure vaults for credential storage. Rotate secrets regularly and avoid hardcoding credentials.
  • Network Security: Implement network segmentation and firewall rules. Monitor for unauthorized network access attempts.

Performance Optimization

  • Query Optimization: Review and optimize frequently-used queries. Use indexed fields and appropriate time ranges.
  • Capacity Planning: Monitor resource utilization trends. Plan for growth and scale proactively.
  • Caching Strategies: Implement caching for frequently-accessed data. Configure cache TTLs appropriately.
  • Index Management: Ensure proper indexing for search performance. Review and optimize index configurations.
  • Data Lifecycle: Implement appropriate retention policies. Archive or delete data according to requirements.
  • Load Distribution: Distribute workloads across available resources. Implement load balancing for high availability.

Comprehensive Troubleshooting

Systematic Troubleshooting Methodology

  1. Define the Problem: Clearly articulate what is not working as expected. Gather specific error messages, timestamps, and affected components.
  2. Gather Information: Collect relevant logs, metrics, and configuration details. Document recent changes that might be related.
  3. Analyze Data: Review collected information for patterns and anomalies. Compare against baseline behavior and known-good configurations.
  4. Develop Hypothesis: Form theories about potential root causes. Prioritize based on likelihood and impact.
  5. Test Solutions: Implement fixes in a controlled manner. Test one change at a time to isolate the effective solution.
  6. Document Resolution: Record the root cause and solution. Update runbooks and knowledge base for future reference.
