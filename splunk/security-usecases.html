Authentication Use Cases

UC-AUTH-001: Brute Force Detection

USE CASE: Brute Force Authentication Attack MITRE: T1110 - Brute Force DATA MODEL: Authentication DETECTION LOGIC: | tstats summariesonly=true count from datamodel=Authentication where Authentication.action=failure by Authentication.user, Authentication.src, _time span=5m | where count > 10 | join type=inner Authentication.user [| tstats summariesonly=true count from datamodel=Authentication where Authentication.action=success by Authentication.user, Authentication.src, _time span=15m] | table _time, Authentication.user, Authentication.src, count CORRELATION SEARCH CONFIG: ├── Schedule: Every 5 minutes ├── Time Range: -15 minutes ├── Throttling: Per user, per src, 1 hour └── Notable Event: Yes, Severity High TUNING CONSIDERATIONS: ├── Exclude service accounts ├── Exclude known vulnerability scanners ├── Adjust threshold based on environment └── Consider geographic anomalies

UC-AUTH-002: Impossible Travel

USE CASE: Impossible Travel Detection MITRE: T1078 - Valid Accounts DATA MODEL: Authentication DETECTION LOGIC: | tstats summariesonly=true latest(_time) as latest_time earliest(_time) as earliest_time values(Authentication.src) as sources dc(Authentication.src) as source_count from datamodel=Authentication where Authentication.action=success by Authentication.user | where source_count > 1 | eval time_diff_hours = (latest_time - earliest_time) / 3600 | where time_diff_hours < 2 | iplocation sources | eval distance = ... (calculate geographic distance) | where distance > 500 AND time_diff_hours < (distance / 500) | table user, sources, time_diff_hours, distance IMPLEMENTATION NOTES: ├── Requires GeoIP lookup ├── VPN connections may cause false positives ├── Consider excluding mobile users └── Integrate with identity provider data

UC-AUTH-003: Password Spray

USE CASE: Password Spray Attack MITRE: T1110.003 - Password Spraying DATA MODEL: Authentication DETECTION LOGIC: | tstats summariesonly=true dc(Authentication.user) as user_count count as attempt_count from datamodel=Authentication where Authentication.action=failure by Authentication.src, _time span=10m | where user_count > 10 AND attempt_count > 20 | eval spray_ratio = user_count / attempt_count | where spray_ratio > 0.5 | table _time, Authentication.src, user_count, attempt_count, spray_ratio KEY INDICATORS: ├── Many unique users targeted ├── Single or few source IPs ├── Short time window ├── Similar failure reasons (wrong password) └── Often targets common usernames

Endpoint Use Cases

UC-END-001: Suspicious PowerShell

USE CASE: Suspicious PowerShell Execution MITRE: T1059.001 - PowerShell DATA MODEL: Endpoint DETECTION LOGIC: | tstats summariesonly=true count from datamodel=Endpoint.Processes where Processes.process_name="powershell.exe" OR Processes.process_name="pwsh.exe" by Processes.dest, Processes.user, Processes.process, Processes.parent_process | search Processes.process IN ("*-enc*", "*-encodedcommand*", "*downloadstring*", "*invoke-expression*", "*iex*", "*bypass*", "*hidden*", "*noprofile*") | table _time, Processes.dest, Processes.user, Processes.process, Processes.parent_process SEVERITY MATRIX: ├── Encoded + Download: Critical ├── Encoded only: High ├── Download only: Medium ├── Bypass/Hidden: Medium └── Base64 in command: High EXCLUSIONS: ├── Known IT management tools ├── Software deployment systems ├── Signed Microsoft scripts └── Documented automation

UC-END-002: Living Off the Land

USE CASE: LOLBin Abuse Detection MITRE: T1218 - System Binary Proxy Execution DATA MODEL: Endpoint DETECTION LOGIC: | tstats summariesonly=true count from datamodel=Endpoint.Processes where Processes.process_name IN ("certutil.exe", "mshta.exe", "regsvr32.exe", "rundll32.exe", "wmic.exe", "cscript.exe", "wscript.exe", "bitsadmin.exe", "msiexec.exe", "installutil.exe", "regasm.exe", "regsvcs.exe") by Processes.dest, Processes.user, Processes.process_name, Processes.process, Processes.parent_process | search Processes.process IN ("*http*", "*ftp*", "*/c*", "*-e*", "*-decode*", "*script:*", "*dll:*", "*//e:*") | table _time, Processes.dest, Processes.user, Processes.process_name, Processes.process LOLBIN-SPECIFIC PATTERNS: ├── certutil: -decode, -urlcache, -split ├── mshta: http://, javascript:, vbscript: ├── regsvr32: /s /n /u /i:http ├── rundll32: javascript:, http:// ├── wmic: /node:, process call create └── bitsadmin: /transfer, /addfile

UC-END-003: Persistence Mechanisms

USE CASE: Registry Persistence Detection MITRE: T1547.001 - Registry Run Keys DATA MODEL: Endpoint DETECTION LOGIC: | tstats summariesonly=true count from datamodel=Endpoint.Registry where Registry.registry_path IN ("*\\CurrentVersion\\Run*", "*\\CurrentVersion\\RunOnce*", "*\\CurrentVersion\\RunServices*", "*\\Winlogon\\Shell*", "*\\Winlogon\\Userinit*") AND Registry.action IN ("created", "modified") by Registry.dest, Registry.user, Registry.registry_path, Registry.registry_value_name, Registry.registry_value_data | table _time, Registry.dest, Registry.user, Registry.registry_path, Registry.registry_value_name, Registry.registry_value_data OTHER PERSISTENCE LOCATIONS: ├── Scheduled Tasks: EventCode=4698 ├── Services: EventCode=7045 ├── Startup Folder: file creation monitoring ├── WMI Subscriptions: WMI event logs └── DLL Search Order: process monitoring

Network Use Cases

UC-NET-001: Beaconing Detection

USE CASE: C2 Beaconing Detection MITRE: T1071 - Application Layer Protocol DATA MODEL: Network_Traffic DETECTION LOGIC: index=firewall OR index=proxy | bin _time span=1h | stats count as connections dc(dest_port) as unique_ports avg(bytes_out) as avg_bytes stdev(bytes_out) as stdev_bytes by src_ip, dest_ip, _time | where connections > 20 AND unique_ports < 3 AND stdev_bytes < (avg_bytes * 0.2) | table src_ip, dest_ip, connections, avg_bytes, stdev_bytes BEACONING INDICATORS: ├── Regular time intervals (±10% variance) ├── Consistent packet sizes ├── Low port diversity ├── Persistent over hours/days └── Unusual destination (rare domain/IP) ADVANCED DETECTION: | stats count by src_ip, dest_ip, _time span=1m | streamstats window=60 avg(count) as avg_rate stdev(count) as std_rate by src_ip, dest_ip | where std_rate < 1 // Very consistent rate

UC-NET-002: Data Exfiltration

USE CASE: Large Data Transfer Detection MITRE: T1048 - Exfiltration Over Alternative Protocol DATA MODEL: Network_Traffic DETECTION LOGIC: | tstats summariesonly=true sum(All_Traffic.bytes_out) as total_bytes from datamodel=Network_Traffic by All_Traffic.src, All_Traffic.dest, _time span=1h | where total_bytes > 100000000 // 100 MB | lookup threat_intel dest as All_Traffic.dest OUTPUT threat_score | where threat_score > 50 OR isnotnull(threat_score) | table _time, All_Traffic.src, All_Traffic.dest, total_bytes, threat_score EXFILTRATION INDICATORS: ├── Unusual upload volume ├── Transfer to cloud storage ├── DNS tunneling (high query volume) ├── After-hours transfers └── Compressed/encrypted streams

UC-NET-003: Lateral Movement

USE CASE: Internal Lateral Movement MITRE: T1021 - Remote Services DATA MODEL: Network_Traffic DETECTION LOGIC: | tstats summariesonly=true count dc(All_Traffic.dest) as dest_count from datamodel=Network_Traffic where All_Traffic.dest_port IN (445, 135, 3389, 5985, 5986, 22) AND All_Traffic.src_category="workstation" by All_Traffic.src, _time span=1h | where dest_count > 5 | table _time, All_Traffic.src, dest_count LATERAL MOVEMENT PATTERNS: ├── SMB (445): File shares, PsExec ├── RPC (135): WMI, DCOM ├── RDP (3389): Remote desktop ├── WinRM (5985/5986): PowerShell remoting ├── SSH (22): Linux/Unix access └── Workstation-to-workstation unusual

Interview Questions

  1. Requirements: Identify threat, map to MITRE, identify data sources
  2. Development: Design detection logic, write SPL query, create notable event
  3. Deployment: Test in dev, tune thresholds, deploy to production
  4. Operations: Monitor FP rate, review quarterly, retire when outdated

Look for multiple failed authentications followed by success:

  • >10 failed logins in 5 minutes from same source
  • Followed by successful login
  • Use Authentication data model with tstats
  • Exclude service accounts and known scanners

Brute Force: Many passwords against one user

Password Spray: One password against many users

Password spray is harder to detect per-user but visible when looking at source IP targeting many users.

  • Look for regular time intervals between connections
  • Consistent packet sizes (low standard deviation)
  • Low port diversity
  • Persistent connections over hours/days
  • Unusual or rare destination IPs/domains
