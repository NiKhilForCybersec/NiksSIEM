Forwarders

Forwarders are the data collection agents of Splunk. Universal Forwarders are lightweight collectors, while Heavy Forwarders provide parsing and routing capabilities.

Forwarder Types

TypeFootprintCapabilitiesUse Case
Universal Forwarder~50MBCollect, forward onlyEndpoints, servers
Heavy Forwarder~1GB+Parse, filter, route, indexSyslog aggregation, routing
Intermediate ForwarderVariesLoad balancing tierScale collection layer

Universal Forwarder Configuration

inputs.conf - Data Collection
# Monitor Windows Security logs
[WinEventLog://Security]
disabled = 0
index = wineventlog
sourcetype = WinEventLog:Security

# Monitor a file
[monitor:///var/log/application.log]
disabled = 0
index = application
sourcetype = app_logs

# Monitor directory
[monitor:///var/log/apache2/]
disabled = 0
index = web
sourcetype = apache:access
whitelist = access\.log$
outputs.conf - Forwarding
# Forward to indexers with load balancing
[tcpout]
defaultGroup = indexers

[tcpout:indexers]
server = idx1.company.com:9997, idx2.company.com:9997, idx3.company.com:9997
compressed = true
useACK = true

# SSL encryption
[tcpout:indexers]
sslCertPath = $SPLUNK_HOME/etc/auth/client.pem
sslRootCAPath = $SPLUNK_HOME/etc/auth/ca.pem
sslVerifyServerCert = true

Heavy Forwarder Use Cases

Syslog Aggregation

Receive syslog from network devices, parse, route to appropriate indexes

Data Filtering

Drop unwanted events before they reach indexers (save license)

Index Routing

Route different sourcetypes to different indexes based on content

Data Masking

Redact sensitive data (SSN, credit cards) before indexing

Interview Questions

UF load balancing in outputs.conf:

  • List multiple indexers: server = idx1:9997, idx2:9997, idx3:9997
  • UF switches between indexers every autoLBFrequency seconds (default 30)
  • If indexer unavailable, automatically fails over to next
  • useACK = true ensures data isn't lost during failover

Deployment Server manages forwarder configurations:

  • Central location for apps/configs
  • Forwarders connect as "deployment clients"
  • Server classes group forwarders (by OS, function, location)
  • Push apps to specific server classes
  • Forwarders poll for updates (default every 60 seconds)
