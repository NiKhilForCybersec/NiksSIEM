Apps & Technology Add-ons

Splunk's app ecosystem includes Technology Add-ons (TAs) for data ingestion and Apps for functionality. Learn to install, configure, and manage them.

Apps vs TAs

TypePurposeContainsInstall On
Technology Add-on (TA)Data ingestion & normalizationinputs, props, transforms, lookupsForwarders, Indexers, Search Heads
AppUser interface & functionalityDashboards, views, searches, alertsSearch Heads only
Supporting Add-on (SA)Shared knowledge objectsLookups, macros, data modelsSearch Heads

TA Deployment Strategy

Where to Install

  • Forwarders: TAs with inputs.conf (data collection)
  • Heavy Forwarders: TAs with props/transforms (parsing)
  • Indexers: TAs with props/transforms if parsing at index time
  • Search Heads: All TAs (for search-time field extraction) + Apps

Interview Questions

Technology Add-on (TA):

  • Handles data ingestion and normalization
  • Contains inputs.conf, props.conf, transforms.conf
  • Maps data to CIM for ES compatibility
  • Deployed across architecture (forwarders, indexers, search heads)

App:

  • Provides user interface and functionality
  • Contains dashboards, views, saved searches
  • Search head only deployment
  • May depend on TAs for data
