Frozen buckets are NOT searchable! This is a common exam question. By default, frozen data is deleted. Set coldToFrozenDir to archive instead.

2.3 Boolean Operator Precedence

Precedence Order (Highest to Lowest)

() → NOT → AND → OR

Trap: error OR warning status=500 evaluates as error OR (warning AND status=500)

Use parentheses: (error OR warning) status=500

Domain 2 Practice Questions

Q1: Which time modifier represents "midnight yesterday"?
  • A) -1d
  • B) -1d@d
  • C) @d-1d
  • D) yesterday
✓ Correct: B) -1d@d
-1d@d goes back 1 day, then snaps to midnight. Without @d, you get exactly 24 hours ago.
Q2: What is the Boolean operator precedence (highest first)?
  • A) AND, OR, NOT, ()
  • B) (), NOT, AND, OR
  • C) OR, AND, NOT, ()
  • D) (), AND, OR, NOT
✓ Correct: B) (), NOT, AND, OR
Parentheses are evaluated first, then NOT, then AND, then OR (lowest).
Q3: Which wildcard search is MOST efficient?
  • A) *error*
  • B) *error
  • C) error*
  • D) All equally efficient
✓ Correct: C) error*
Trailing wildcards use the index efficiently. Leading wildcards require full scans.

3.2 Field Commands

Essential Field Commands

| fields a, b, cINCLUDE only these fields (faster)
| fields - a, bEXCLUDE these fields
| table a, b, cFormat output as table
| rename old AS newRename fields
| dedup userRemove duplicates (keeps first)
| dedup user sortby -_timeKeep MOST RECENT per user
| sort -amountSort descending (- prefix)
| sort 10 -amountTop 10 by amount
fields vs table
  • fields - Improves performance by reducing data early in pipeline
  • table - Formats output but doesn't improve performance
  • Use fields early, table at the end

4.2 search vs where

Featuresearchwhere
Wildcards✓ Yes✗ No
Field-to-field comparison✗ No✓ Yes
Equality operator=== (double equals)
Functions✗ No✓ Yes
When to Use Each
  • search: Use for wildcards (status=fail*)
  • where: Use for field comparisons (where sent > received)
  • where: Use for functions (where len(user) > 10)

Final Exam Tips

  • High-weight domains: Basic Searching (22%) + Fields (20%) + SPL (17%) + Transforming (15%) = 74%
  • Time modifiers: earliest=-1d@d latest=@d for complete yesterday
  • search vs where: Wildcards with search, field-to-field with where
  • dedup sortby -_time: Keeps most recent per unique value
  • dc() not count(): Use dc() for unique counts
  • Frozen = NOT searchable: Common exam trap!
