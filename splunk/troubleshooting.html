Splunk Troubleshooting

Diagnose and resolve common Splunk issues. Use btool, internal logs, and health checks to identify problems.

Diagnostic Tools

ToolPurposeUsage
btoolCheck effective configuration./splunk btool inputs list --debug
diagCreate support bundle./splunk diag
Monitoring ConsoleHealth dashboardsSettings > Monitoring Console
_internal indexSplunk's own logsindex=_internal

btool Commands

Configuration Troubleshooting
# Check inputs configuration
./splunk btool inputs list --debug

# Check props.conf settings for a sourcetype
./splunk btool props list syslog --debug

# Check transforms
./splunk btool transforms list --debug

# Find configuration conflicts
./splunk btool inputs list --debug | grep -A2 "monitor://"

# Check outputs configuration
./splunk btool outputs list --debug

Internal Log Searches

Troubleshooting Searches
# Indexing errors
index=_internal sourcetype=splunkd log_level=ERROR

# Forwarder connectivity issues
index=_internal sourcetype=splunkd "connection" "refused OR failed OR error"

# License usage
index=_internal sourcetype=license_usage_summary

# Search performance issues
index=_internal sourcetype=scheduler status=* | stats count by status

# Data ingestion rates
index=_internal sourcetype=splunkd group=per_index_thruput

# Search head clustering issues
index=_internal sourcetype=splunkd component=CMMaster

Common Issues

Data Not Indexing

Check inputs.conf, permissions, forwarder connectivity, license limits

Slow Searches

Check time range, use tstats, add index/sourcetype, optimize SPL

Time Parsing Issues

Check TIME_FORMAT, TIME_PREFIX in props.conf, verify timestamps

License Violations

Check daily volume, identify high-volume sources, consider filtering

Interview Questions

  1. Check source: Is data being generated? Check file exists, permissions
  2. Check forwarder: Is UF running? Check splunkd.log for errors
  3. Check inputs.conf: Use btool to verify configuration is loaded
  4. Check connectivity: Can forwarder reach indexers? Test port 9997
  5. Check indexer: Is data being received? index=_internal source=*metrics*
  6. Check license: Are you over license limit?
  7. Check index: Is data going to expected index? Try index=*
