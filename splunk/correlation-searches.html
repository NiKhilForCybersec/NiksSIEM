Correlation Searches

Correlation searches are Splunk ES's detection engine. They run scheduled SPL queries that create notable events when threats are detected.

How Correlation Searches Work

1

Scheduled Execution

SPL query runs on schedule (e.g., every 5 minutes)

2

Query Returns Results

If query finds matching events, results are passed to actions

3

Notable Event Created

Results create notable events in Incident Review

4

Adaptive Response

Optional automated actions (email, script, risk modifier)

Correlation Search Example

Brute Force Detection
| tstats summariesonly=true count from datamodel=Authentication
  where Authentication.action=failure
  by Authentication.user, Authentication.src, _time span=5m
| rename Authentication.* as *
| where count > 10
| eval rule_name="Brute Force Attack Detected"
| eval severity="high"
| collect index=notable
