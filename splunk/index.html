Quick Start SPL Queries

SPL Failed Authentication Events 
index=security sourcetype=WinEventLog:Security EventCode=4625
| stats count by user, src_ip
| where count > 5
| sort - count
SPL Process Execution Timeline 
index=endpoint sourcetype=sysmon EventCode=1
| eval process_path=mvindex(split(Image,"\\"),-1)
| timechart span=1h count by process_path
| sort - count
SPL Network Traffic Analysis 
index=network sourcetype=firewall action=blocked
| stats sum(bytes) as total_bytes, count as connection_count by src_ip, dest_ip, dest_port
| eval total_mb=round(total_bytes/1024/1024,2)
| where connection_count > 100
| table src_ip, dest_ip, dest_port, connection_count, total_mb

Interview Preparation

Explain the difference between Universal and Heavy Forwarders.
Key Differences:

Universal Forwarder (UF) is lightweight (~50MB RAM), sends raw data without processing, and is ideal for most endpoints and servers. Heavy Forwarder (HF) consumes more resources, can parse and filter data at source, route to multiple destinations, and perform data masking. Use UF for simple log collection, HF when you need preprocessing, data routing, or custom transformations before indexing.

What is risk-based alerting in Splunk Enterprise Security?
Risk-Based Alerting (RBA):

RBA aggregates risk scores from multiple detections on assets and identities rather than alerting on each individual event. When an entity accumulates risk above a threshold, a notable event is created. Benefits include: Reduced alert fatigue by consolidating related detections, better context showing all contributing factors, and prioritized response focusing on highest-risk entities. Implementation requires properly configured Asset and Identity frameworks.
