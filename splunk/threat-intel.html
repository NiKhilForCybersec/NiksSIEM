Threat Intelligence

Integrate threat intelligence feeds into Splunk Enterprise Security. Match IOCs against logs, configure threat intel lookups, and enrich events.

Threat Intel Framework

How ES Uses Threat Intel

Enterprise Security includes a Threat Intelligence framework that downloads, normalizes, and matches IOCs against your data. When a match is found, ES can create notable events or add threat context to existing events.

Threat Intel Searches

IOC Matching Queries
# Match IPs against threat intel
index=network 
| lookup threat_intel_ip ip as dest_ip OUTPUT threat_type, confidence, description
| where isnotnull(threat_type)
| table _time, src_ip, dest_ip, threat_type, confidence, description

# Match domains in DNS logs
index=dns 
| lookup threat_intel_domain domain as query OUTPUT threat_type, threat_category
| where isnotnull(threat_type)
| stats count by query, threat_type, threat_category

# Match file hashes
index=endpoint sourcetype=sysmon EventCode=1
| lookup threat_intel_hash hash as SHA256 OUTPUT malware_family, threat_level
| where isnotnull(malware_family)
| table _time, Computer, SHA256, malware_family, threat_level

# Enrich all traffic with threat context
index=proxy 
| lookup local_threat_intel ip as dest_ip OUTPUT threat_score, threat_tags
| eval enriched = if(isnotnull(threat_score), "yes", "no")

Interview Questions

Matching methods:

  • Lookup tables: TI data stored in KV store, searched via lookup command
  • Correlation searches: Scheduled queries that join threat intel with events
  • Automatic lookups: Configured in props.conf to enrich at search time
  • Real-time matching: Streaming searches for immediate alerting

IOC types matched: IP addresses, domains, URLs, file hashes, email addresses
