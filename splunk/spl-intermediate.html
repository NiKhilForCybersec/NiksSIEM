SPL Intermediate

Build on SPL fundamentals with subsearches, transactions, lookups, and advanced eval functions for complex security analysis.

Subsearches

Subsearch Examples
// Find events from top 10 talkers
index=network sourcetype=firewall
    [search index=network sourcetype=firewall earliest=-1h
    | stats sum(bytes) as total_bytes by src_ip
    | sort -total_bytes
    | head 10
    | fields src_ip]

// Events from users in HR department (lookup-based)
index=auth 
    [| inputlookup employees.csv 
    | search department="HR" 
    | fields user]

Transactions

Transaction Command
// Group events into sessions by user
index=web sourcetype=access_log
| transaction user maxpause=30m
| table user, duration, eventcount

// Group login to logout
index=auth
| transaction user startswith="action=login" endswith="action=logout"
| table user, duration, eventcount

// Multi-event attack chain
index=security
| transaction src_ip maxspan=1h maxpause=5m
| search eventcount > 5
| table src_ip, duration, eventcount

Lookups

Lookup Commands
// Enrich with lookup
index=auth
| lookup employees.csv user OUTPUT department, manager, location
| table _time, user, department, action

// GeoIP lookup
index=firewall
| iplocation src_ip
| stats count by src_ip, Country, City

// Threat intel lookup
index=proxy
| lookup threat_intel.csv domain AS dest_host OUTPUT threat_type, confidence
| where isnotnull(threat_type)

Interview Questions

transaction: When you need to see related events as a single item, preserve raw events, calculate duration. Memory-intensive.

stats: When you only need aggregations (count, sum, values). Much faster and efficient.

Rule: Use stats when possible, transaction only when you need event grouping with duration/eventcount.

  • CSV lookups: Static file-based, good for reference data
  • KV Store: NoSQL database, faster for large datasets
  • External lookups: Python scripts for dynamic data
  • Automatic lookups: Applied automatically at search time
