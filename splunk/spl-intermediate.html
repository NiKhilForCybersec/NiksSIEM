SPL Intermediate

Build on SPL fundamentals with subsearches, transactions, lookups, macros, and eval functions for complex security analysis.

Subsearches

Subsearch Examples
# Find events from top talkers
index=network 
[search index=network | top 10 src_ip | fields src_ip]

# Find users who logged in from known bad IPs
index=authentication action=success
[| inputlookup bad_ips.csv | fields ip | rename ip as src]

# Events from users with failed logins
index=* 
[search index=authentication action=failure 
| stats count by user 
| where count > 5 
| fields user]

Transactions

Transaction Examples
# Group events by session
index=web 
| transaction session_id maxpause=5m

# Login to logout session
index=authentication 
| transaction user startswith="action=login" endswith="action=logout" maxspan=8h
| eval session_duration = duration
| table user, session_duration, eventcount

# Multi-stage attack (recon -> exploit -> exfil)
index=security 
| transaction src_ip maxspan=1h 
| search eventcount > 3

Lookups

Lookup Examples
# Enrich with asset information
index=security 
| lookup asset_lookup ip as src_ip OUTPUT asset_name, asset_criticality, owner
| table _time, src_ip, asset_name, asset_criticality, event_type

# Check against threat intel
index=network 
| lookup threat_intel_ip ip as dest_ip OUTPUT threat_type, confidence
| where isnotnull(threat_type)

# Automatic lookup (configured in transforms.conf)
# Just query - enrichment happens automatically
index=authentication
| table user, department, manager  # department/manager from identity lookup

Advanced Eval Functions

Eval Examples
# Multiple conditions with case
| eval severity = case(
    count > 100, "critical",
    count > 50, "high",
    count > 10, "medium",
    true(), "low"
)

# Coalesce - first non-null value
| eval user = coalesce(src_user, dest_user, "unknown")

# String manipulation
| eval domain = mvindex(split(email, "@"), 1)
| eval filename = mvindex(split(file_path, "\\"), -1)

# Time functions
| eval hour = strftime(_time, "%H")
| eval day_of_week = strftime(_time, "%A")
| eval is_business_hours = if(hour >= 9 AND hour <= 17, "yes", "no")

# Regex extraction
| rex field=message "user=(?[^\s]+)"
| rex field=url "https?://(?[^/]+)"

Interview Questions

Subsearch:

  • Runs first, returns values used in main search
  • Limited to ~10,000 results, 60 seconds
  • Good for: filtering main search by dynamic list

Join:

  • Combines fields from two datasets
  • More flexible but memory-intensive
  • Good for: enrichment, correlation between datasets

Best practice: Use subsearch for filtering, join for enrichment, stats for aggregation-based correlation
