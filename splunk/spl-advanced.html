tstats - Accelerated Searches

Why tstats?

tstats queries accelerated data models and tsidx files - 10-100x faster than regular searches. Essential for ES correlation searches.

tstats Examples
// Count authentication failures (accelerated)
| tstats summariesonly=true count from datamodel=Authentication 
  where Authentication.action=failure 
  by Authentication.user, Authentication.src

// Network traffic summary
| tstats summariesonly=true sum(Network_Traffic.bytes) as total_bytes 
  from datamodel=Network_Traffic 
  by Network_Traffic.src_ip, Network_Traffic.dest_ip
  
// Time-based aggregation
| tstats summariesonly=true count from datamodel=Authentication
  where Authentication.action=* 
  by _time span=1h, Authentication.action
| timechart span=1h count by Authentication.action

Macros

Macro Definition & Usage
// Define in macros.conf:
[get_failures(1)]
args = threshold
definition = index=auth action=failure | stats count by user | where count > $threshold$

// Usage:
\`get_failures(10)\`

// With time argument:
[security_events(2)]
args = time, sourcetype
definition = index=security sourcetype=$sourcetype$ earliest=-$time$

Search Optimization

Time First

Always specify time range. Use earliest/latest before index.

Filter Early

Put most restrictive filters first. Index > sourcetype > fields.

Use tstats

Query accelerated data models instead of raw events.

Limit Fields

Use fields command to reduce data transferred.

Interview Questions

  1. Time range: Narrow earliest/latest as much as possible
  2. Index specification: Always specify index=
  3. Filtering order: Index > sourcetype > indexed fields > extracted fields
  4. Use tstats: Query data models instead of raw
  5. Avoid wildcards: Especially leading wildcards (*error)
  6. Fields command: Reduce fields early
  7. Summary indexing: Pre-aggregate frequent searches

Purpose: Pre-compute and store summarized data for fast queries

How: Enable acceleration on data model, Splunk builds tsidx summaries

Query with: tstats command with summariesonly=true

Trade-off: Uses storage space, requires rebuild time
