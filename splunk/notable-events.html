Notable Events

Notable events are the core alert mechanism in Splunk Enterprise Security. Correlation searches generate notables, which analysts triage in the Incident Review dashboard.

Notable Event Status Workflow

New
  • Just created, awaiting triage
  • SLA clock starts
In Progress
  • Analyst investigating
  • Assigned owner
Pending
  • Waiting on external input
  • SLA may pause
Closed
  • Resolution documented
  • Disposition set (TP/FP)

Interview Questions

  1. Review false positives to identify patterns
  2. Add exclusions to the search (whitelist users, hosts)
  3. Adjust thresholds (count, time window)
  4. Add contextual conditions (asset type, user group)
  5. Consider converting to risk-based alerting
