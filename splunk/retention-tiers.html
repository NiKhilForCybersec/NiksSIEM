Data Retention

Splunk stores data in buckets that move through lifecycle stages. Understanding hot/warm/cold/frozen buckets is key for capacity planning.

Bucket Lifecycle

Bucket Aging Process
Hot
Active writes
→
Warm
Searchable, read-only
→
Cold
Archive storage
→
Frozen
Delete or archive
BucketStorageSearchableWritableTransition Trigger

FrozenDelete/ArchiveNo*NoRetention policy

indexes.conf Settings

Retention Configuration
[security]
homePath = $SPLUNK_DB/security/db
coldPath = $SPLUNK_DB/security/colddb
thawedPath = $SPLUNK_DB/security/thaweddb

# Retention settings
frozenTimePeriodInSecs = 7776000    # 90 days
maxTotalDataSizeMB = 500000         # 500 GB max

# Hot/Warm settings  
maxHotBuckets = 10
maxDataSize = auto_high_volume
maxWarmDBCount = 300

Interview Questions

  1. Copy frozen bucket from archive to thawedPath
  2. Splunk automatically rebuilds index files
  3. Data becomes searchable in "thawed" state
  4. Remove when done to reclaim space

Alternatively, use SmartStore with remote storage for automatic tiering.
