Broker VM - Complete Setup & Configuration Guide

Step-by-step guide to deploying, configuring, and managing Broker VMs for on-premises log collection. Includes actual console paths, configuration commands, and production-ready syslog configurations.

Step 1: Deploy Broker VM

Download Broker VM Image

XSIAM Console → Settings → Data Sources → Broker VM → + Add Broker VM
1
Navigate to Broker VM Setup

Settings (gear icon) → Data Sources → Broker VM → Click "+ Add Broker VM"

2
Download OVA Image

Select "Download OVA" for VMware or "Download VHD" for Hyper-V

Image size: ~8-10 GB. Keep the registration token - you'll need it.

3
Copy Registration Token

Copy the token shown - this links the VM to your XSIAM tenant

# Token format example: eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJodHRwczovL2FwaS5wYWxv...

VMware vSphere Deployment

4
Import OVA

vSphere Client → Right-click cluster/host → Deploy OVF Template → Select downloaded OVA

5
Configure VM Resources
Small (5K EPS):4 vCPU, 16 GB RAM, 500 GB disk
Medium (15K EPS):8 vCPU, 32 GB RAM, 1 TB disk
Large (40K EPS):16 vCPU, 64 GB RAM, 2 TB disk
6
Configure Networking

Assign to network that can:

  • Receive syslog from log sources (typically internal/DMZ)
  • Reach *.paloaltonetworks.com:443 (XSIAM cloud)

Step 2: Initial Broker VM Configuration

Console Access & Registration

1
Access VM Console

Open VM console in vSphere. Default credentials:

Username: admin Password: admin # You'll be forced to change password on first login
2
Configure Network (Static IP Recommended)
# From Broker VM CLI menu, select: Network Configuration broker-vm> configure network Interface: eth0 IP Address: 10.0.1.50 Subnet Mask: 255.255.255.0 Gateway: 10.0.1.1 DNS Server 1: 10.0.1.10 DNS Server 2: 10.0.1.11 # Apply and verify broker-vm> show network
3
Register with XSIAM
broker-vm> register # Paste the registration token from XSIAM console Token: eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9... # Wait for "Registration successful" message # Broker VM will appear in XSIAM console within 2-5 minutes
4
Verify Registration
broker-vm> show status # Expected output: Registration Status: Registered Cloud Connectivity: Connected Last Check-in: 2024-02-15 14:32:01 UTC Certificate Status: Valid

Step 3: Configure Syslog Data Sources

Add Syslog Listener in XSIAM Console

Settings → Data Sources → Add Data Source → Syslog Collector
1
Create Syslog Data Source

Data Sources → + Add Data Source → Select "Syslog Collector"

2
Configure Listener Settings
Name: paloalto-firewall-syslog
Broker VM: Select your registered Broker VM
Protocol: TCP (recommended) or UDP
Port: 514 (or custom port)
Vendor: Palo Alto Networks
Product: Firewall
3
Save and Deploy

Click Save. Configuration pushes to Broker VM within 1-2 minutes.

Configure Source Device (Palo Alto Firewall Example)

Firewall GUI → Device → Server Profiles → Syslog

Step A: Create Syslog Server Profile

# Firewall GUI: Device → Server Profiles → Syslog → Add Name: XSIAM-Broker Servers: Name: broker-vm-1 Server: 10.0.1.50 # Your Broker VM IP Transport: TCP Port: 514 Format: BSD Facility: LOG_USER

Step B: Create Log Forwarding Profile

# Objects → Log Forwarding → Add Name: Forward-to-XSIAM Match List: # Entry 1: Traffic logs Name: traffic-logs Log Type: traffic Filter: All Logs # Or filter to reduce volume Syslog: XSIAM-Broker # Entry 2: Threat logs Name: threat-logs Log Type: threat Filter: All Logs Syslog: XSIAM-Broker # Entry 3: URL logs Name: url-logs Log Type: url Filter: All Logs Syslog: XSIAM-Broker # Entry 4: Authentication logs Name: auth-logs Log Type: auth Filter: All Logs Syslog: XSIAM-Broker

Step C: Apply Log Forwarding to Security Rules

# Policies → Security → Edit each rule Actions tab: Log Setting: Log at Session End # Required to generate logs Log Forwarding: Forward-to-XSIAM # Apply the profile # Commit changes

Step D: Configure System Logs (Optional but Recommended)

# Device → Log Settings → System System Logs: Syslog: XSIAM-Broker # Sends config changes, admin actions Configuration Logs: Syslog: XSIAM-Broker # Sends commit/config events

! Configure logging to Broker VM logging enable logging timestamp logging host inside 10.0.1.50 tcp/514 logging trap informational logging facility 20 ! Enable specific log messages (security relevant) logging class auth trap informational logging class access trap informational logging class ha trap warnings logging class ids trap informational ! Verify configuration show logging

Step 4: Verify Data is Flowing

Check Broker VM Status

# SSH or console to Broker VM broker-vm> show status Registration Status: Registered Cloud Connectivity: Connected Last Upload: 2024-02-15 14:35:22 UTC # Check listener status broker-vm> show listeners Listener Protocol Port Status Events/sec -------------------------------------------------------- syslog-tcp-514 TCP 514 Active 1,247 syslog-udp-514 UDP 514 Active 832 wef-collector WinRM 5985 Active 156

Verify in XSIAM Console

Settings → Data Sources → [Your Data Source] → Status

Check that "Last Event Received" shows recent timestamp and "Events/Hour" shows expected volume.

Query Data in XQL

// Check Palo Alto firewall data dataset = panw_ngfw_traffic_raw | filter _time > ago(1h) | comp count = count() | fields count // Check by source IP (firewall management IP) dataset = panw_ngfw_traffic_raw | filter _collector_source = "10.0.1.100" // Firewall IP | limit 10 // Check Windows events dataset = microsoft_windows_raw | filter _time > ago(1h) | comp count = count() by event_id | sort desc count | limit 20

Troubleshooting Common Issues

Issue Diagnostic Command Resolution
Broker VM not appearing in console broker-vm> show status Re-run registration. Check HTTPS 443 outbound to *.paloaltonetworks.com
No events from syslog source tcpdump -i eth0 port 514 Verify source is sending. Check firewall rules on Broker VM network.
Events received but not in XSIAM broker-vm> show upload-queue Check cloud connectivity. Queue backup indicates upload issues.
Data not parsing correctly Check raw logs in XQL Verify correct Vendor/Product selected in data source config.
High disk usage broker-vm> show disk Upload backlog. Check bandwidth or cloud connectivity.

Interview Questions

Complete end-to-end process:

  1. Deploy Broker VM:
    • Download OVA from XSIAM console (Settings → Data Sources → Broker VM)
    • Deploy to VMware with appropriate sizing (8 vCPU/32GB for ~15K EPS)
    • Configure static IP on network accessible by firewalls
    • Register with XSIAM using provided token
  2. Create Syslog Data Source in XSIAM:
    • Settings → Data Sources → Add → Syslog Collector
    • Select Broker VM, Protocol (TCP), Port (514)
    • Set Vendor (Palo Alto Networks) and Product (Firewall)
  3. Configure Firewall:
    • Create Syslog Server Profile pointing to Broker VM IP
    • Create Log Forwarding Profile for Traffic, Threat, URL logs
    • Apply Log Forwarding Profile to security rules
    • Commit configuration
  4. Verify:
    • Check Broker VM: show listeners shows events/sec
    • Query XQL: dataset = panw_ngfw_traffic_raw | limit 10

High disk = upload backlog. Systematic troubleshooting:

  1. Check queue status:
    broker-vm> show upload-queue

    If queue is large, data isn't uploading to cloud.

  2. Test cloud connectivity:
    broker-vm> connectivity test

    Verify HTTPS 443 to *.paloaltonetworks.com works.

  3. Check bandwidth:

    10K EPS ≈ 10-20 Mbps sustained upload needed. Check for throttling.

  4. Check for proxy issues:

    If using proxy, verify it's configured correctly and not blocking.

  5. Verify EPS vs capacity:
    broker-vm> show listeners

    If incoming EPS exceeds VM capacity, need to scale up or add another Broker VM.

Emergency mitigation: If disk fills, oldest logs drop. Priority is restoring cloud connectivity.
