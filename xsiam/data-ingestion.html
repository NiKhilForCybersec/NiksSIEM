Cortex XDR Agent Data

Agent Data Collection

Data Collected by Agent: Process Events: - Process creation/termination - Command line arguments - Parent-child relationships - User context - Hash values File Events: - File create/modify/delete - File path and attributes - Hash on creation - Signature information Network Events: - Connection attempts - Source/destination IPs and ports - Process making connection - Protocol information Registry Events (Windows): - Key create/delete - Value set/modify - Persistence locations monitored Additional Data: - Module loads - DNS queries - Script content - Memory operations

Syslog Collection

Settings → Data Sources → Syslog Collector

Syslog Collector Configuration

Syslog Collector Setup: 1. Deploy Broker VM - Download from XSIAM console - Deploy in network accessible to log sources - Configure network settings 2. Configure Collection - Protocol: TCP/UDP - Port: 514 (default) or custom - TLS: Enable for secure transport 3. Create Parsing Rules - Map to vendor/product - Define log format - Set dataset destination 4. Point Sources - Configure devices to send to collector - Verify connectivity - Check for data flow

AWS Integration

Data Normalization

XDM (XSIAM Data Model)

XDM Normalization: Purpose: - Standardize field names across sources - Enable cross-source correlation - Consistent query experience Common XDM Fields: ├── xdm.source.ipv4 ├── xdm.target.ipv4 ├── xdm.source.user.username ├── xdm.target.host.hostname ├── xdm.event.type ├── xdm.event.outcome └── xdm.alert.severity Mapping Process: 1. Raw data ingested 2. Parsing rules applied 3. Fields mapped to XDM 4. Normalized data stored

Custom Parsing Rules

Settings → Data Sources → Parsing Rules

Creating Custom Parser: 1. Identify Log Format - Sample logs - Field delimiters - Timestamp format 2. Create Parsing Rule - Regex extraction - Field mapping - XDM normalization 3. Test Parser - Use sample data - Verify field extraction - Check XDM mapping 4. Deploy - Apply to data source - Monitor for errors

Storage and Retention

Storage Tiers

XSIAM Storage Architecture: HOT Storage: - Recent data (configurable) - Fast query performance - Full functionality COLD Storage: - Older data - Archived for compliance - Slower queries - Rehydration needed for analysis Retention Settings: - Per dataset configuration - Based on compliance needs - Cost optimization

Troubleshooting Data Issues

Common Issues: No Data Arriving: 1. Check source connectivity 2. Verify collector status 3. Check parsing rules 4. Review firewall rules Parsing Failures: 1. Review raw logs 2. Check parser configuration 3. Update regex patterns 4. Test with sample data Missing Fields: 1. Verify source configuration 2. Check field extraction 3. Update XDM mapping

Interview Questions

  1. Identify data source: What type, what data needed?
  2. Check native support: Built-in parser available?
  3. Configure collection: API, syslog, or agent
  4. Set up parsing: Map fields to XDM
  5. Test data flow: Verify in Query Center
  6. Create detections: Build correlation rules

XDM (XSIAM Data Model): Standardized field naming schema

  • Correlation: Join data from different sources
  • Consistent queries: One query works for all data
  • Analytics: ML models work across sources
