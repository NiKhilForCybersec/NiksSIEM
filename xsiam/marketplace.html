Marketplace Overview

Understanding the content ecosystem in XSIAM

What is the Marketplace?

The XSIAM Marketplace (also known as Content Hub) is a centralized repository for downloadable content that extends platform capabilities. Unlike building custom integrations from scratch, the marketplace provides tested, version-controlled content maintained by Palo Alto Networks, partners, and the community.

Version Control

All content packs use semantic versioning (MAJOR.MINOR.PATCH). Breaking changes increment MAJOR, new features increment MINOR, and bug fixes increment PATCH. Dependencies can specify version ranges.

Installing & Managing Content

How to deploy and maintain content packs

Installation Methods

1

Marketplace UI (Recommended)

Navigate to Settings â†’ Content â†’ Marketplace. Search for the pack, click Install, and configure any required parameters. Dependencies are automatically resolved.

2

CLI Installation (demisto-sdk)

For CI/CD pipelines and bulk operations, use the demisto-sdk command line tool to upload packs programmatically.

CLI Pack Installation Bash 
# Install demisto-sdk
pip install demisto-sdk
# Configure connection
export DEMISTO_BASE_URL="https://your-xsiam-instance.xdr.us.paloaltonetworks.com"
export DEMISTO_API_KEY="your-api-key"
# Upload a content pack
demisto-sdk upload -i Packs/MyPack
# Install from marketplace
demisto-sdk marketplace install -i VirusTotal
# List installed packs
demisto-sdk marketplace list
# Update all packs
demisto-sdk marketplace update --all
3

Custom Pack Upload

For internal or modified content, package as a zip file and upload via Settings â†’ Content â†’ Custom Content.

Handling Conflicts

  • Version Conflicts: If two packs require different versions of a dependency, the higher compatible version is used
  • Circular Dependencies: Detected and rejected during pack validation
  • Breaking Changes: Major version updates may require manual intervention

Customization Warning

If you modify a pack's content (playbooks, scripts), your changes may be overwritten during updates. Either fork the pack as custom content or track modifications in version control for re-application.

Creating Custom Content

Building and publishing your own content packs

Development Workflow

1. Setup Development Environment

Install demisto-sdk, configure IDE (VS Code recommended), and clone the content repository. Use demisto-sdk init to scaffold a new pack.

2. Develop Integration/Content

Write Python code following XSOAR conventions. Use CommonServerPython helpers. Implement required commands and outputs.

3. Write Unit Tests

Create test files with pytest. Mock external API calls. Aim for >80% code coverage. Run with demisto-sdk test.

4. Validate Content

Run demisto-sdk validate to check structure, schemas, and best practices. Fix any errors before proceeding.

5. Test in Dev Instance

Upload to development XSIAM instance. Create test playbooks. Verify all commands work with real data.

6. Document

Write README.md with setup instructions. Add command examples. Document any prerequisites or limitations.

7. Package and Deploy

Run demisto-sdk create-content-artifact to create uploadable zip. Deploy to production or submit to community.

Integration Development Example

Creating a basic integration with the standard patterns:

MyIntegration.py Python 
import demistomock as demisto
from CommonServerPython import *
from CommonServerUserPython import *
import requests
# Disable insecure warnings
requests.packages.urllib3.disable_warnings()
class Client(BaseClient):
    """
    Client class for API interactions.
    Inherits from BaseClient which handles auth, retries, etc.
    """
    def __init__(self, base_url: str, api_key: str, verify: bool, proxy: bool):
        super().__init__(
            base_url=base_url,
            verify=verify,
            proxy=proxy,
            headers={'Authorization': f'Bearer {api_key}'}
        )
    def get_indicator(self, indicator: str) -> dict:
        """Get indicator details from API."""
        return self._http_request(
            method='GET',
            url_suffix=f'/indicators/{indicator}'
        )
    def search_indicators(self, query: str, limit: int = 50) -> list:
        """Search indicators matching query."""
        return self._http_request(
            method='POST',
            url_suffix='/search',
            json_data={'query': query, 'limit': limit}
        )
def test_module(client: Client) -> str:
    """Test API connectivity."""
    try:
        client.get_indicator('test')
        return 'ok'
    except DemistoException as e:
        raise DemistoException(f'Test failed: {str(e)}')
def get_indicator_command(client: Client, args: dict) -> CommandResults:
    """Command to get indicator details."""
    indicator = args.get('indicator')
    result = client.get_indicator(indicator)
    # Map to standard context
    outputs = {
        'Value': result.get('indicator'),
        'Type': result.get('type'),
        'Score': result.get('risk_score'),
        'Malicious': result.get('risk_score', 0) > 70
    }
    # Calculate DBot score
    dbot_score = Common.DBotScore(
        indicator=indicator,
        indicator_type=DBotScoreType.IP if result.get('type') == 'ip' else DBotScoreType.URL,
        integration_name='MyIntegration',
        score=3 if outputs['Malicious'] else 1
    )
    return CommandResults(
        outputs_prefix='MyIntegration.Indicator',
        outputs_key_field='Value',
        outputs=outputs,
        indicator=dbot_score,
        readable_output=tableToMarkdown('Indicator Results', outputs)
    )
def main():
    params = demisto.params()
    args = demisto.args()
    command = demisto.command()
    client = Client(
        base_url=params.get('url'),
        api_key=params.get('api_key'),
        verify=not params.get('insecure', False),
        proxy=params.get('proxy', False)
    )
    demisto.debug(f'Command: {command}')
    try:
        if command == 'test-module':
            return_results(test_module(client))
        elif command == 'myintegration-get-indicator':
            return_results(get_indicator_command(client, args))
        else:
            raise NotImplementedError(f'Command {command} not implemented')
    except Exception as e:
        demisto.error(traceback.format_exc())
        return_error(f'Failed: {str(e)}')
if __name__ in ('__main__', '__builtin__', 'builtins'):
    main()

ğŸ’¼ Interview Answer: Content Strategy

Question: "How would you approach building an XSOAR content strategy for a new deployment?"

  • Audit existing tools: List all security tools that need integration - prioritize by alert volume and response criticality
  • Check marketplace first: For each tool, check if certified/partner content exists - avoid building from scratch
  • Identify gaps: Document which integrations need custom development or modification
  • Start with use cases: Don't install everything - start with 2-3 key use cases (e.g., phishing, malware)
  • Build incrementally: Get basic playbooks working, then add enrichment and edge cases over time
  • Document and version: Track all customizations in Git, document deviations from standard packs
  • Plan for maintenance: Assign owners for content updates, schedule quarterly reviews
