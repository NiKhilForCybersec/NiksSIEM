XSIAM Marketplace

The Marketplace provides pre-built content packs for integrations, playbooks, and detections. This guide covers how to find, install, configure, and manage content packs.

What is the Marketplace?

Definition

The Marketplace is XSIAM's app store for security content. It contains pre-built integrations, playbooks, detection rules, and dashboards created by Palo Alto and the community.

What Content Packs Include

Integrations

Connect to 3rd party tools (ServiceNow, Splunk, etc.)

Playbooks

Automation workflows for XSOAR

Detection Rules

Pre-built correlation rules

Dashboards

Pre-configured visualizations

Where to Find the Marketplace

XSIAM Console → Marketplace (left navigation)

Marketplace Sections:

Marketplace → Browse Search and discover available content packs
Marketplace → Installed View and manage installed packs
Marketplace → Updates Available updates for installed packs
Marketplace → Private Upload custom content packs

How to Install a Content Pack

Step-by-Step Installation

Marketplace → Browse → [Search] → Install
1
Search for Content Pack

Marketplace → Browse → Use search bar or filters

Example: Search "ServiceNow" to find ServiceNow integration

2
Review Pack Details

Click on pack to view:

  • Description and use cases
  • Included integrations, playbooks, rules
  • Dependencies (other packs required)
  • Version history
3
Click Install

Click the "Install" button

Dependencies will be auto-installed if required

4
Configure the Integration

After installation, configure credentials:

Settings → Integrations → [Integration Name] → + Add Instance

Example: Installing & Configuring ServiceNow

ServiceNow

Integration Playbooks ITSM

Step 1: Install the Pack

Marketplace → Search "ServiceNow" → Click "ServiceNow v2" → Install

Step 2: Configure Integration Instance

Settings → Integrations → ServiceNow v2 → + Add Instance
Required Configuration Fields:
Instance URL: https://yourcompany.service-now.com
Username: xsiam_integration (service account)
Password: [Service account password]
Client ID/Secret: (Optional - for OAuth)

Step 3: Test the Integration

Click "Test" button to verify connectivity

✓ Success message confirms integration is working

Step 4: Use in Playbooks

The integration adds commands like:

  • !servicenow-create-ticket - Create incident ticket
  • !servicenow-update-ticket - Update existing ticket
  • !servicenow-query-tickets - Search tickets

Managing Content Pack Updates

Marketplace → Updates

Update Process:

1
Check for Updates

Updates tab shows packs with new versions available

2
Review Change Log

Click pack → View "What's New" to see changes

⚠️ Breaking changes may require playbook updates

3
Update

Click "Update" → Confirm

Existing instance configurations are preserved

Best Practice

Schedule monthly content pack reviews. Update in a dev/test tenant first if you have one.

Interview Questions

Step-by-step process:

  1. Find the content pack: Marketplace → Browse → Search for the vendor/product
  2. Install: Click on pack → Review dependencies → Click Install
  3. Configure instance: Settings → Integrations → [Integration] → + Add Instance
  4. Enter credentials: URL, API key, username/password as required
  5. Test: Click "Test" to verify connectivity
  6. Use in playbooks: The integration's commands are now available in playbook builder

Example: To add VirusTotal - install the pack, create instance with API key, test, then use !file command in playbooks to check file hashes.

Essential packs for any deployment:

  • Core: Common Scripts, Common Playbooks (dependencies for everything)
  • Use Cases: Phishing, Malware, Endpoint Investigation
  • Enrichment: VirusTotal, URLhaus, AbuseIPDB
  • Identity: Active Directory, Azure AD (user lookups)
  • Ticketing: ServiceNow or Jira (ITSM integration)
  • Communication: Slack or Microsoft Teams (notifications)

Why these: They cover the core workflow - detect threat, enrich with context, identify affected users, create ticket, notify analysts.
