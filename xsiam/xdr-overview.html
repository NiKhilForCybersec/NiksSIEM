Detection Types in XSIAM

BIOC Rules (Behavioral Indicators of Compromise)

BIOC rules are the core endpoint detection mechanism in Cortex XDR. They detect malicious behaviors based on process execution, file operations, network connections, and registry modifications.

BIOC Rule Categories

Process BIOC

Detects suspicious process behaviors including execution chains, command-line arguments, and parent-child relationships.

File BIOC

Monitors file system operations for indicators of malicious activity, ransomware, or data exfiltration.

Network BIOC

Detects suspicious network connections, beaconing patterns, and C2 communication attempts.

Registry BIOC

Monitors registry modifications commonly used for persistence and privilege escalation.

Analytics BIOC (ML-Powered Detection)

Analytics BIOC uses machine learning to detect threats without predefined signatures. It learns normal behavior baselines and identifies anomalies that may indicate compromise.

Answer: Analytics BIOC learns normal behavior patterns over a baselining period (typically 2-4 weeks). It establishes baselines for user behavior, process execution, network connections, and file access patterns. After baselining, it generates alerts for deviations from normal behavior. The baselining period should capture typical business operations including any cyclical patterns.
