Dashboards & Reporting

XSIAM's dashboard and reporting capabilities provide real-time visibility into security operations, enabling analysts and executives to monitor threats, track KPIs, and demonstrate security posture.

Built-in Dashboard Types

Incident Dashboard

Real-time view of active incidents, severity distribution, and analyst workload

Alert Dashboard

Alert trends, top triggered rules, and alert-to-incident conversion rates

Endpoint Dashboard

Agent health, protection status, and endpoint threat landscape

Network Dashboard

Traffic analysis, blocked connections, and network threat indicators

Identity Dashboard

Authentication trends, risky users, and access anomalies

ASM Dashboard

External attack surface, discovered assets, and risk scores

Creating Custom Dashboards

Dashboard Builder Interface

Navigation: Investigation → Dashboards → Create Dashboard

  • Layout: Drag-and-drop widget placement with resizable panels
  • Widgets: Charts, tables, single values, maps, and custom XQL
  • Filters: Global time range and entity filters
  • Refresh: Auto-refresh intervals (1min to 24hr)
  • Sharing: Share with users, roles, or make public

Dashboard XQL Queries

SOC Operations Widgets

Open Incidents by Severity
dataset = xdr_incidents
| filter status != "Resolved" and status != "Closed"
| comp count() as count by severity
| sort desc count
Mean Time to Resolution (MTTR)
dataset = xdr_incidents
| filter status in ("Resolved", "Closed")
| filter _time > ago(30d)
| alter resolution_time = timestamp_diff(
    resolved_timestamp, creation_timestamp, "HOUR")
| comp avg(resolution_time) as avg_mttr_hours,
    median(resolution_time) as median_mttr_hours
| alter avg_mttr_hours = round(avg_mttr_hours, 1),
    median_mttr_hours = round(median_mttr_hours, 1)
Alert Trend (24-Hour)
dataset = xdr_alerts
| filter _time > ago(24h)
| comp count() as alerts by _time = truncate(_time, 1h)
| sort asc _time
Top Triggered Detection Rules
dataset = xdr_alerts
| filter _time > ago(7d)
| comp count() as alert_count,
    count_distinct(xdm.source.host.hostname) as affected_hosts
    by rule_name
| sort desc alert_count
| limit 10

Endpoint Security Widgets

Agent Health Status
dataset = endpoints
| comp count() as count by agent_status
| alter status_label = if(agent_status = "CONNECTED", "Healthy",
    if(agent_status = "DISCONNECTED", "Offline", "Degraded"))
Malware Blocked (Last 7 Days)
dataset = xdr_data
| filter xdm.event.type = "MALWARE"
| filter xdm.event.outcome = XDM_CONST.OUTCOME_BLOCKED
| filter _time > ago(7d)
| comp count() as blocked_count,
    count_distinct(xdm.source.host.hostname) as affected_endpoints
    by xdm.alert.name
| sort desc blocked_count
| limit 10

Network Security Widgets

Blocked Connections by Category
dataset = xdr_data
| filter xdm.observer.product = "Firewall"
| filter xdm.event.outcome = XDM_CONST.OUTCOME_BLOCKED
| filter _time > ago(24h)
| comp count() as blocked by xdm.network.application_category
| sort desc blocked
| limit 10
Geographic Attack Origins
dataset = xdr_alerts
| filter _time > ago(7d)
| filter xdm.source.location.country != null
| filter xdm.source.ipv4 not incidr "10.0.0.0/8"
| comp count() as attacks by 
    xdm.source.location.country,
    xdm.source.location.latitude,
    xdm.source.location.longitude
| sort desc attacks
| limit 20

SOC Metrics & KPIs

Metric Definition Target XQL Calculation
MTTD Mean Time to Detect < 1 hour avg(detection_time - event_time)
MTTR Mean Time to Respond < 4 hours avg(resolution_time - creation_time)
MTTA Mean Time to Acknowledge < 15 min avg(assignment_time - creation_time)
FP Rate False Positive Rate < 20% count(fp) / count(total) * 100
Alert Volume Alerts per day Varies count() by day
Incidents/Analyst Workload distribution < 10/day count() by assigned_to
Complete SOC KPI Dashboard Query
// All key metrics in one query
dataset = xdr_incidents
| filter _time > ago(30d)
| alter 
    mtta_hours = timestamp_diff(assigned_timestamp, creation_timestamp, "HOUR"),
    mttr_hours = timestamp_diff(resolved_timestamp, creation_timestamp, "HOUR"),
    is_fp = if(resolution = "False Positive", 1, 0),
    is_resolved = if(status in ("Resolved", "Closed"), 1, 0)
| comp 
    count() as total_incidents,
    sum(is_resolved) as resolved_incidents,
    sum(is_fp) as false_positives,
    avg(mtta_hours) as avg_mtta,
    avg(mttr_hours) as avg_mttr,
    median(mttr_hours) as median_mttr
| alter 
    resolution_rate = multiply(divide(resolved_incidents, total_incidents), 100),
    fp_rate = multiply(divide(false_positives, resolved_incidents), 100),
    avg_mtta = round(avg_mtta, 1),
    avg_mttr = round(avg_mttr, 1),
    resolution_rate = round(resolution_rate, 1),
    fp_rate = round(fp_rate, 1)

Dashboard Sharing & Access

Access Control Options

  • Private: Only creator can view
  • Shared with Users: Specific user access
  • Shared with Roles: All users with specific role
  • Public: All XSIAM users can view
⚠️ Dashboard Best Practices
  • Performance: Limit widgets to 10-12 per dashboard
  • Time Ranges: Use appropriate ranges (don't query years of data)
  • Refresh Rates: Match refresh to data urgency (real-time vs historical)
  • Naming: Use clear, descriptive dashboard names
  • Documentation: Add descriptions to explain widget purpose

Interview Answer: "What metrics do you track on a SOC dashboard?"

"I structure SOC dashboards around key operational and strategic metrics. For operational awareness, I track open incidents by severity, alert volume trends, and analyst workload distribution. For performance measurement, I monitor MTTD (Mean Time to Detect), MTTR (Mean Time to Respond), and false positive rates by detection rule. For executives, I create summary dashboards showing security posture trends, top threats mitigated, and MITRE ATT&CK coverage. The key is tailoring dashboards to the audience - analysts need actionable details, managers need workload visibility, and executives need trend-level insights. In XSIAM, I use XQL queries to calculate these metrics and create widgets that auto-refresh at appropriate intervals."
