XSIAM Dashboards - Complete Configuration Guide

Build effective SOC dashboards with XQL-powered widgets. This guide covers dashboard creation, widget types, XQL queries for each visualization, and production-ready SOC dashboard examples.

Dashboard Concepts

How XSIAM Dashboards Work

XSIAM dashboards are built from widgets. Each widget runs an XQL query and displays results in a visualization (chart, table, number, etc.). Dashboards refresh on schedule or manually.

Creating a Dashboard

XSIAM Console → Dashboards → + New Dashboard
1
Create New Dashboard

Dashboards (left nav) → + New Dashboard → Enter name (e.g., "SOC Operations")

2
Set Dashboard Properties
Name:SOC Operations Dashboard
Refresh Interval:5 minutes (or manual)
Default Time Range:Last 24 hours
Shared:Yes (visible to team)
3
Add Widgets

Click "+ Add Widget" → Select visualization type → Enter XQL query

Widget Types & XQL Queries

1. Number Widget (Single Stat)

Open Critical Incidents

Shows a single number prominently - perfect for KPIs

// XQL Query for Number Widget dataset = xdr_incident_raw | filter status != "RESOLVED" and status != "CLOSED" | filter severity = "HIGH" or severity = "CRITICAL" | comp count = count()
23
Open Critical Incidents

2. Pie/Donut Chart

Incidents by Severity

Shows distribution across categories

// XQL Query for Pie Chart dataset = xdr_incident_raw | filter _time > ago(24h) | comp count = count() by severity

Widget Config: Group By = severity, Value = count

3. Bar Chart

Top 10 Alert Sources

Shows ranked comparison

// XQL Query for Bar Chart dataset = xdr_data | filter _time > ago(24h) | filter event_type = ALERT | comp alert_count = count() by agent_hostname | sort desc alert_count | limit 10

4. Line Chart (Time Series)

Alert Volume Over Time

Shows trends over time - essential for spotting anomalies

// XQL Query for Time Series dataset = xdr_data | filter _time > ago(7d) | filter event_type = ALERT | comp alert_count = count() by bin(_time, 1h) | sort asc _time

Key: Use bin(_time, interval) to bucket time. Options: 1h, 4h, 1d, etc.

5. Table Widget

Recent High-Severity Alerts

Shows detailed records

// XQL Query for Table dataset = xdr_incident_raw | filter _time > ago(24h) | filter severity in ("HIGH", "CRITICAL") | fields _time, incident_id, name, severity, status, assigned_to | sort desc _time | limit 20

Production SOC Dashboard Example

Complete SOC Operations Dashboard

Copy these XQL queries to build a comprehensive SOC dashboard:

Widget 1: Open Incidents (Number)

dataset = xdr_incident_raw | filter status in ("NEW", "UNDER_INVESTIGATION") | comp count = count()

Widget 2: MTTR - Mean Time to Respond (Number)

// Calculate MTTR for incidents closed in last 7 days dataset = xdr_incident_raw | filter status = "RESOLVED" and resolved_time > ago(7d) | alter response_time_hours = timestamp_diff(resolved_time, creation_time, "HOUR") | comp avg_mttr = avg(response_time_hours) | alter avg_mttr = round(avg_mttr, 1)

Widget 3: Incidents by Severity (Pie Chart)

dataset = xdr_incident_raw | filter _time > ago(24h) | comp count = count() by severity

Widget 4: Top Targeted Hosts (Bar Chart)

dataset = xdr_data | filter _time > ago(24h) | filter event_type = ALERT | comp alerts = count() by agent_hostname | sort desc alerts | limit 10

Widget 5: Alert Trend (Line Chart - 7 Days)

dataset = xdr_data | filter _time > ago(7d) | filter event_type = ALERT | comp count = count() by bin(_time, 4h), severity | sort asc _time

Widget 6: Top MITRE Techniques Detected (Bar Chart)

dataset = xdr_incident_raw | filter _time > ago(30d) | filter mitre_technique_id != null | comp count = count() by mitre_technique_id, mitre_technique_name | sort desc count | limit 10

Widget 7: Firewall Blocked Connections (Line Chart)

dataset = panw_ngfw_traffic_raw | filter _time > ago(24h) | filter action = "deny" | comp blocked = count() by bin(_time, 1h) | sort asc _time

Widget 8: Failed Logins by User (Table)

dataset = xdr_data | filter _time > ago(24h) | filter event_type = AUTH and action_outcome = FAILURE | comp failed_logins = count() by xdm.target.user.username, xdm.source.ipv4 | filter failed_logins >= 5 | sort desc failed_logins | limit 20

Executive Dashboard Widgets

Security Posture Score (Number)

// Calculate score based on open incidents and coverage dataset = xdr_incident_raw | filter status in ("NEW", "UNDER_INVESTIGATION") | comp critical = countif(severity = "CRITICAL"), high = countif(severity = "HIGH"), medium = countif(severity = "MEDIUM") | alter score = 100 - (critical * 10) - (high * 5) - (medium * 2) | alter score = if(score < 0, 0, score) | fields score

Month-over-Month Incident Trend

dataset = xdr_incident_raw | filter _time > ago(60d) | alter month = format_timestamp("%Y-%m", _time) | comp incidents = count() by month | sort asc month

Endpoint Coverage (Number)

dataset = xdr_agents | filter agent_status = "CONNECTED" | comp protected_endpoints = count()

Interview Questions

I'd create a SOC Operations Dashboard with these key widgets:

  1. Open Incidents (Number): Query incidents where status != RESOLVED
  2. MTTR (Number): Calculate average time between creation and resolution using timestamp_diff()
  3. Alert Volume Trend (Line): Use bin(_time, 1h) to bucket alerts over time
  4. Incidents by Severity (Pie): Group by severity field
  5. Top Targeted Hosts (Bar): Count alerts grouped by agent_hostname
  6. Analyst Workload (Table): Count incidents by assigned_to field

Configuration: Set dashboard refresh to 5 minutes, default time range to 24 hours, and share with SOC team.

Use the bin() function to bucket time:

dataset = xdr_data | filter _time > ago(7d) | comp count = count() by bin(_time, 1h) | sort asc _time

Key points:

  • bin(_time, interval) - Groups timestamps into buckets
  • Common intervals: 15m, 1h, 4h, 1d
  • Always sort by _time ascending for proper chart display
  • Add a second group-by field for multiple series (e.g., by severity)
