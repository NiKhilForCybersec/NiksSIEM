Threat Intelligence Management (TIM)

XSIAM's built-in Threat Intelligence Management provides enrichment, correlation, and actionable context from Unit 42 research and third-party feeds to enhance detection and response.

What is TIM in XSIAM?

Threat Intelligence Management (TIM) is the native capability to collect, normalize, analyze, and operationalize threat intelligence data. XSIAM includes:

  • Unit 42 Intelligence - Premium threat research from Palo Alto's dedicated team
  • AutoFocus Tags - Malware family and campaign classification
  • Third-Party Feeds - STIX/TAXII, CSV, JSON imports
  • Custom Indicators - Your own organizational IoCs

Memory Trick: "TIM = Threat Info Made-actionable"

TIM takes raw threat data and makes it actionable by enriching alerts, correlating across datasets, and providing context for investigation.

Confidence Scoring

XSIAM assigns confidence scores to indicators to help prioritize alerts:

● HIGH Confidence (80-100)

Verified malicious with multiple confirmations. Suitable for automated blocking.

  • Unit 42 confirmed C2 infrastructure
  • Multiple vendor consensus
  • Recent sandbox detection
  • Active campaign attribution

● MEDIUM Confidence (40-79)

Likely malicious but requires verification. Good for alerting.

  • Single vendor report
  • Historical association
  • Heuristic detection
  • Aged but unverified IoC

● LOW Confidence (1-39)

Suspicious activity, use for hunting and context only.

  • Open-source unverified
  • Very old indicators
  • Generic suspicious patterns
  • Community submissions

XQL Queries for Threat Intel

Find Alerts Matching Known Threat Actors

XQL 
dataset = xdr_alerts
| join type = inner (
    dataset = threat_intelligence_indicators
    | filter threat_actor != null
    | fields indicator_value, threat_actor, confidence
) as ti on dst_ip = ti.indicator_value or src_ip = ti.indicator_value
| filter ti.confidence >= 70
| fields alert_name, src_ip, dst_ip, ti.threat_actor, ti.confidence, timestamp
| sort desc timestamp

Enrich Incidents with Intel Context

XQL 
dataset = xdr_incidents
| arrayexpand network_artifacts as ip
| join type = left (
    dataset = threat_intelligence_indicators
    | fields indicator_value, indicator_type, source, tags, first_seen, last_seen
) as intel on ip = intel.indicator_value
| filter intel.indicator_value != null
| fields incident_id, description, ip, intel.source, intel.tags, intel.first_seen
| dedup incident_id, ip

Hunt for Specific Malware Family

XQL 
// Get all Emotet-related indicators
dataset = threat_intelligence_indicators
| filter tags contains "Emotet" or malware_family = "Emotet"
| fields indicator_type, indicator_value, confidence, last_seen
// Then hunt in network logs
dataset = firewall_logs
| join type = inner (
    dataset = threat_intelligence_indicators
    | filter malware_family = "Emotet"
    | fields indicator_value
) as emotet on dst_ip = emotet.indicator_value or url contains emotet.indicator_value
| fields timestamp, src_ip, dst_ip, url, action

Dashboard: Indicator Statistics

XQL 
dataset = threat_intelligence_indicators

Sample XSOAR Enrichment Script

Python (XSOAR) 
def enrich_indicator(indicator_value, indicator_type):
    """
    Enriches an indicator with threat intelligence from multiple sources.
    """
    enrichment = {
        'indicator': indicator_value,
        'type': indicator_type,
        'sources': []
    }
    # Unit 42 lookup
    unit42_result = demisto.executeCommand('unit42-intel-lookup', {
        'indicator': indicator_value
    })
    if unit42_result and unit42_result[0].get('Contents'):
        enrichment['sources'].append({
            'name': 'Unit 42',
            'data': unit42_result[0]['Contents']
        })
    # VirusTotal lookup
    vt_result = demisto.executeCommand('vt-ip-report' if indicator_type == 'ip' else 'vt-hash-report', {
        'resource': indicator_value
    })
    if vt_result and vt_result[0].get('Contents'):
        enrichment['sources'].append({
            'name': 'VirusTotal',
            'data': vt_result[0]['Contents']
        })
    # Calculate overall score
    enrichment['confidence_score'] = calculate_confidence(enrichment['sources'])
    enrichment['recommended_action'] = get_recommendation(enrichment['confidence_score'])
    return enrichment
def calculate_confidence(sources):
    """Calculate weighted confidence score from multiple sources."""
    weights = {'Unit 42': 1.0, 'VirusTotal': 0.8, 'OpenSource': 0.5}
    total_weight = 0
    weighted_score = 0
    for source in sources:
        name = source['name']
        score = source['data'].get('confidence', 50)
        weight = weights.get(name, 0.5)
        weighted_score += score * weight
        total_weight += weight
    return weighted_score / total_weight if total_weight > 0 else 0

TIM Best Practices

Feed Hygiene

Regularly review and expire old indicators. Set appropriate TTLs based on indicator type.

Layered Sources

Combine commercial (high quality) with open source (broad coverage) feeds.

Confidence Thresholds

Only auto-block on high confidence. Alert on medium. Hunt on low.

Sector-Specific Intel

Join your industry ISAC for targeted intelligence relevant to your sector.

Interview Answer: "How do you operationalize threat intelligence?"

"Operationalizing threat intelligence means making it actionable in your security operations. In XSIAM, I configure multiple intel sources - Unit 42 as our primary high-confidence feed, plus ISAC feeds for sector-specific threats, and open-source feeds like Abuse.ch for breadth. Each indicator gets a confidence score based on source reliability, age, and corroboration. High-confidence indicators (80+) are pushed to prevention controls like firewalls. Medium confidence triggers alerts that analysts review. Low confidence goes into a hunting database. We use XSOAR playbooks to automatically enrich every alert with intel context - if an IP in an alert matches a known C2 server, the analyst immediately sees the threat actor, associated malware family, and TTPs. This context dramatically speeds up triage decisions."

Interview Answer: "What is STIX/TAXII?"

"STIX (Structured Threat Information eXpression) is a standardized language for describing cyber threat intelligence - things like indicators, threat actors, campaigns, and TTPs. TAXII (Trusted Automated eXchange of Indicator Information) is the transport protocol for sharing STIX data. Think of STIX as the format and TAXII as the delivery mechanism. In XSIAM, we configure TAXII clients to poll servers like our ISAC's threat feed, automatically ingesting STIX 2.1 bundles containing indicators. The standardization means we can consume intel from any STIX-compliant source without custom parsing. XSIAM normalizes these into its native indicator format for correlation against alerts."
