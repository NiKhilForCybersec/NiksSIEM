Threat Intelligence - Complete Configuration Guide

How to configure, import, and operationalize threat intelligence in XSIAM. Covers IOC management, feed integration, and using TI in detection rules.

What is Threat Intelligence in XSIAM?

Definition

Threat Intelligence (TI) in XSIAM consists of Indicators of Compromise (IOCs) that are matched against your logs to detect known threats. When network traffic, file hashes, or URLs match a known-bad indicator, an alert is generated.

IOC Types Supported

SHA256 Hash SHA1 Hash MD5 Hash IPv4 Address IPv6 Address Domain URL Email

How to Import IOCs Manually

Method 1: CSV Upload

Threat Intel → Import → Upload File
1
Prepare CSV File

Create CSV with required columns:

indicator,type,severity,expiration,description 185.220.101.1,IP,HIGH,2024-12-31,Tor exit node - C2 traffic e99a18c428cb38d5f260853678922e03,MD5,CRITICAL,2024-12-31,Emotet loader malware-c2.evil.com,DOMAIN,HIGH,2024-12-31,APT29 C2 domain
2
Upload in Console

Threat Intel → Import → Choose File → Select your CSV

Map columns to fields if needed

3
Verify Import

Go to Threat Intel → Indicators → Search for your IOCs

IOCs are now active and will match against incoming data

Using Threat Intel in XQL Queries

Query 1: Find All IOC Matches in Last 24 Hours

dataset = threat_intel_matches | filter _time > ago(24h) | fields _time, indicator_value, indicator_type, severity, matched_data_source, matched_field, matched_value | sort desc _time

Query 2: Correlate IOC Match with Endpoint Context

// Join TI matches with endpoint data for full context dataset = threat_intel_matches | filter indicator_type = "IP" | join type=inner ( dataset = xdr_data | filter event_type = NETWORK ) as endpoint_data on matched_value = endpoint_data.xdm.target.ipv4 | fields _time, indicator_value, endpoint_data.agent_hostname, endpoint_data.action_process_image_name

Query 3: Check Specific IOC Across All Data

// Search for a specific suspicious IP dataset = xdr_data, panw_ngfw_traffic_raw | filter xdm.target.ipv4 = "185.220.101.1" or xdm.source.ipv4 = "185.220.101.1" | fields _time, xdm.source.ipv4, xdm.target.ipv4, xdm.source.host.hostname, xdm.event.type | sort desc _time

Configuring IOC Matching Rules

Threat Intel → IOC Rules → + New Rule

What Are IOC Rules?

IOC Rules define what happens when an IOC matches. You can configure different actions based on severity, IOC type, or feed source.

Creating an IOC Rule

1
Define Match Criteria
IOC Types: IP, Domain (select which types this rule applies to)
Severity: High, Critical (only match high-confidence IOCs)
Feed: Unit 42, AlienVault (specific feeds only)
2
Define Actions
  • Generate Alert: Create incident in XSIAM (always recommended)
  • Trigger Playbook: Auto-run XSOAR playbook for enrichment/response
  • Block at Firewall: Push block rule to NGFW (if integrated)
  • Isolate Endpoint: Auto-isolate affected host (use carefully!)

Interview Questions

Step-by-step process:

  1. Navigate: Threat Intel → Feeds → + Add Feed
  2. Select feed type: TAXII, STIX, CSV URL, or vendor-specific API
  3. Configure connection: Enter URL, API key, polling interval
  4. Map fields: Ensure indicator types and severity are mapped correctly
  5. Test and save: Verify connectivity, then enable the feed
  6. Create IOC rules: Define what happens when indicators match (alert, playbook, block)

Example: For AlienVault OTX, I'd use TAXII feed type, enter their discovery URL, add my API key, set 1-hour polling, and create a rule to generate High severity alerts on matches.

False positive management strategies:

  1. Use severity filtering: Only alert on High/Critical IOCs, not Low/Medium
  2. Whitelist legitimate indicators: Threat Intel → Indicators → Select → Add to Exclusion List
  3. Evaluate feed quality: Track FP rate per feed, disable low-quality feeds
  4. Require correlation: Don't alert on IOC match alone - require additional context (e.g., IOC match + suspicious process)
  5. Set expiration dates: Old IOCs become stale - ensure feeds include expiration or set default TTL

XQL for analysis:

dataset = threat_intel_matches | filter _time > ago(30d) | comp total = count(), fp = countif(resolution = "FALSE_POSITIVE") by feed_name | alter fp_rate = round(fp / total * 100, 1) | sort desc fp_rate
