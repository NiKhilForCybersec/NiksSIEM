Recon & Attack Surface Management

Cortex Xpanse integration provides continuous discovery and monitoring of your internet-facing assets, identifying risks before attackers do. ASM is the proactive arm of XSIAM security.

What is Attack Surface Management?

Attack Surface Management (ASM) is the continuous process of discovering, classifying, and monitoring all external-facing assets that could be exploited by attackers. In XSIAM, this capability comes from Cortex Xpanse integration.

Memory Trick: "ASM = Always Scanning Myself"

Think of ASM as constantly looking at your organization from an attacker's perspective - always scanning what's exposed to the internet.

Asset Discovery

Cortex Xpanse continuously scans the internet to discover assets belonging to your organization using multiple techniques:

IP Range Scanning

Scans all IPs in your registered ranges and associated ASNs

Certificate Analysis

Finds assets through SSL/TLS certificate organization fields

DNS Enumeration

Discovers subdomains and DNS records for your domains

Cloud Provider APIs

Integrates with AWS, Azure, GCP for cloud asset discovery

Linked Asset Analysis

Traces connections between known and unknown assets

Historical Data

Uses historical internet data to find forgotten assets

Risk Scoring

Xpanse assigns risk scores based on multiple factors:

CRITICAL Score 9-10

Actively exploited vulnerabilities, exposed databases, default credentials on critical systems.

  • Exposed RDP with no MFA
  • Unpatched Log4j instances
  • Open MongoDB without auth
  • Exposed cloud storage with PII

HIGH Score 7-8

Known CVEs, misconfigured services, sensitive services exposed to internet.

  • Outdated Apache/Nginx versions
  • SSH with password auth enabled
  • Admin panels without 2FA
  • Development servers in prod

MEDIUM Score 4-6

Security misconfigurations, missing hardening, non-critical exposures.

  • Missing security headers
  • Weak TLS configurations
  • Directory listing enabled
  • Verbose error messages

LOW Score 1-3

Informational findings, best practice recommendations.

  • Missing HSTS header
  • Cookie without Secure flag
  • Old but patched software
  • Non-critical info disclosure

XQL Queries for ASM Data

Query Critical Risk Assets

XQL 
dataset = xpanse_assets
| filter risk_score >= 9
| fields asset_name, asset_type, ip_address, risk_score, issues, last_seen
| sort desc risk_score
| limit 100

Find Newly Discovered Assets (Last 24 Hours)

XQL 
dataset = xpanse_assets
| filter first_seen >= timestamp_sub(current_time(), "24h")
| fields asset_name, asset_type, ip_address, first_seen, risk_score
| sort desc first_seen

Assets with Known CVEs

XQL 
dataset = xpanse_assets
| filter cve_ids != null
| arrayexpand cve_ids
| join type = left (
    dataset = threat_intel_cve
    | fields cve_id, cvss_score, exploit_available
) as cve_data on cve_ids = cve_data.cve_id
| filter cve_data.exploit_available = true
| fields asset_name, ip_address, cve_ids, cve_data.cvss_score
| sort desc cve_data.cvss_score

Shadow IT Detection (Unmanaged Assets)

XQL 
dataset = xpanse_assets
| filter managed_status = "unmanaged" or managed_status = "unknown"
| filter risk_score >= 5
| fields asset_name, ip_address, asset_type, discovery_method, business_unit
| sort desc risk_score

ASM Best Practices

Scope Definition

Clearly define IP ranges, domains, and cloud accounts to monitor. Include subsidiaries and acquired companies.

Owner Assignment

Map every asset to a business owner. Integrate with CMDB for automatic owner lookup.

SLA Definition

Define remediation SLAs by risk level. Track metrics and escalate overdue issues.

Continuous Monitoring

Enable real-time alerts for new discoveries and configuration changes.

Interview Answer: "What is Attack Surface Management?"

"Attack Surface Management is the practice of continuously discovering and monitoring all internet-facing assets that belong to an organization. In XSIAM, this is powered by Cortex Xpanse which scans the internet using techniques like IP range enumeration, certificate analysis, DNS reconnaissance, and cloud API integration. The key value is finding assets you didn't know existed - shadow IT, forgotten dev servers, or assets from acquisitions. Xpanse assigns risk scores based on CVEs, misconfigurations, and exposure level, then generates alerts that feed into the same incident queue as XDR alerts. This allows security teams to prioritize remediation of external vulnerabilities alongside internal threat detection."

Interview Answer: "How does ASM differ from Vulnerability Scanning?"

"Traditional vulnerability scanning requires you to know what assets exist and point the scanner at them. ASM flips this - it starts from the internet and discovers what's exposed, including assets you don't know about. Vulnerability scanners go deep on known assets; ASM goes wide across your entire external footprint. In an ideal security program, ASM feeds newly discovered assets into your vulnerability management program for deeper scanning. XSIAM's integration means Xpanse discoveries can automatically trigger Tenable, Qualys, or Rapid7 scans via XSOAR playbooks."
