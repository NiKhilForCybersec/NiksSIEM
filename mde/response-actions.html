Response Actions

Take immediate action on threats using MDE's response capabilities. Isolate devices, collect forensic packages, and run live response sessions.

Available Actions

ActionDescriptionImpact
Isolate DeviceCut network except MDE communicationHigh - User loses network
Restrict App ExecutionOnly MS-signed apps can runHigh - May break apps
Run AV ScanTrigger full or quick scanLow - Background task
Collect Investigation PackageGather forensic dataLow - Background collection
Initiate Live ResponseInteractive remote shellMedium - Direct access
Stop & Quarantine FileKill process, quarantine fileMedium - Stops process

Device Isolation

Isolation Considerations

  • Device loses ALL network except MDE service
  • User cannot access network resources, internet
  • VPN connections terminated
  • MDE continues to send telemetry and receive commands
  • Can release isolation when investigation complete

Investigation Package Contents

Package Includes

  • Autoruns: Persistence mechanisms
  • Installed programs: Software inventory
  • Network connections: Active connections at collection time
  • Scheduled tasks: All scheduled tasks
  • Services: Installed services
  • Security event logs: Recent security events
  • Prefetch files: Evidence of program execution

Interview Questions

Use isolation when:

  • Active attacker on device (lateral movement risk)
  • Ransomware detected (prevent spread)
  • C2 communication confirmed
  • Need to preserve evidence while investigating

Consider alternatives when:

  • Single suspicious file - use Stop & Quarantine
  • Need investigation first - collect package
  • Lower severity - run AV scan
  • Business critical system - restrict apps instead
