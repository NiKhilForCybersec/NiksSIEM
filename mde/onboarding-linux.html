Linux Onboarding

Deploy Microsoft Defender for Endpoint on Linux servers. Supports major distributions including RHEL, CentOS, Ubuntu, Debian, SLES, and Oracle Linux.

Supported Distributions

DistributionVersionsPackage
RHEL / CentOS7.2+, 8.x, 9.xRPM (yum/dnf)
Ubuntu18.04+, 20.04, 22.04DEB (apt)
Debian9+, 10, 11DEB (apt)
SLES12+, 15RPM (zypper)
Oracle Linux7.2+, 8.xRPM (yum)

Installation (Ubuntu/Debian)

APT Installation
# 1. Install prerequisites
sudo apt-get install curl apt-transport-https gnupg

# 2. Add Microsoft repository key
curl -sSL https://packages.microsoft.com/keys/microsoft.asc | sudo apt-key add -

# 3. Add repository
sudo apt-add-repository "https://packages.microsoft.com/ubuntu/$(lsb_release -rs)/prod"

# 4. Update and install
sudo apt-get update
sudo apt-get install mdatp

# 5. Download onboarding script from portal and run
sudo python3 MicrosoftDefenderATPOnboardingLinuxServer.py

# 6. Verify
mdatp health
mdatp health --field org_id

Installation (RHEL/CentOS)

YUM Installation
# 1. Add Microsoft repository
sudo yum-config-manager --add-repo=https://packages.microsoft.com/config/rhel/8/prod.repo

# 2. Install
sudo yum install mdatp

# 3. Onboard
sudo python3 MicrosoftDefenderATPOnboardingLinuxServer.py

# 4. Verify
mdatp health
mdatp connectivity test

Common Commands

mdatp CLI
# Check health status
mdatp health

# Check specific field
mdatp health --field real_time_protection_enabled

# Run scan
mdatp scan quick
mdatp scan full
mdatp scan custom --path /home/user

# Threat management
mdatp threat list
mdatp threat quarantine

# Update definitions
mdatp definitions update

# View configuration
mdatp config

# Connectivity test
mdatp connectivity test

Interview Questions

  • Kernel: 3.10.0-327 or later
  • Disk space: 1GB minimum
  • Python: Python 3.x for onboarding script
  • Network: Outbound HTTPS to Microsoft endpoints
  • Package manager: apt, yum, dnf, or zypper
  • fanotify: Kernel support for file monitoring
