KQL Advanced Hunting - Suspicious Process 
DeviceProcessEvents
| where Timestamp > ago(7d)
| where InitiatingProcessFileName =~ "powershell.exe"
| where ProcessCommandLine has_any ("-enc", "-encoded", "bypass", "hidden")
| project Timestamp, DeviceName, AccountName, ProcessCommandLine, InitiatingProcessCommandLine
| limit 100

Interview Preparation

How does Microsoft Defender for Endpoint differ from traditional antivirus?
Key Differences:

MDE is an EDR platform that goes far beyond traditional AV: Full telemetry collection captures process trees, network connections, registry changes, and file operations. Cloud-delivered protection uses Microsoft's threat intelligence and ML models. Behavioral detection identifies threats by actions rather than signatures alone. Automated Investigation and Response (AIR) automatically investigates and remediates threats. Advanced hunting enables proactive threat hunting with KQL. Traditional AV only performs signature-based file scanning without the broader visibility and response capabilities.

What are Attack Surface Reduction (ASR) rules?
ASR Rules Overview:

ASR rules are preventive controls that block specific behaviors commonly used in attacks. Examples include: Block executable content from email prevents weaponized attachments. Block Office apps from creating child processes stops macro-based attacks. Block credential stealing from LSASS prevents credential dumping. Block process creations from WMI/PSExec limits lateral movement. Rules should be deployed in audit mode first to identify false positives before enabling block mode. ASR rules require Windows Defender Antivirus in active mode.
