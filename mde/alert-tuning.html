Alert Tuning

Reduce alert fatigue by tuning MDE detections. Create suppressions, custom detection rules, and optimize alert quality.

Tuning Methods

Suppression Rules

Suppress alerts by file, folder, process, or behavior. Quick fix for known false positives.

Indicator Exclusions

Allow specific files, IPs, URLs, or certificates. Bypasses detection entirely.

Custom Detections

Create your own detection rules using KQL. Tailor to your environment.

Alert Policies

Adjust severity, enable/disable built-in detections per policy.

Suppression Rules

Common Suppressions
# Suppress by file hash (SHA256)
Indicator Type: File hash
Action: Allow
SHA256: abc123...

# Suppress by folder path
Indicator Type: File path
Action: Allow  
Path: C:\Program Files\MyApp\*

# Suppress by certificate
Indicator Type: Certificate
Action: Allow
Thumbprint: 123ABC...
Signer: "Contoso Corporation"

# Suppress by IP (for internal scanners)
Indicator Type: IP Address
Action: Allow
IP: 10.0.0.50

Tuning Workflow

1

Identify False Positives

Review alerts, identify patterns causing false positives

2

Analyze Root Cause

Determine what's triggering - file, behavior, network?

3

Choose Tuning Method

Suppression for specific alerts, exclusion for system-wide

4

Implement Narrowly

Make exclusions as specific as possible

5

Monitor & Validate

Verify FPs reduced without creating blind spots

Interview Questions

  • Be specific: Narrow exclusions (specific path, not C:\*)
  • Document everything: Track why each exclusion exists
  • Review regularly: Quarterly review of all suppressions
  • Prefer suppression over exclusion: Alert still fires, just hidden
  • Use conditions: Combine criteria (hash + path + user)
  • Monitor for abuse: Attackers may target excluded paths
