AD Attack Detection Queries

// Splunk - Password Spray Detection index=windows EventCode=4625 | bucket _time span=5m | stats dc(TargetUserName) as unique_users, values(TargetUserName) as users by src_ip, _time | where unique_users > 10 // KQL - Kerberoasting Detection SecurityEvent | where EventID == 4769 | where ServiceName !endswith "$" | where TicketEncryptionType == "0x17" // RC4 | summarize count() by Account, ServiceName, IpAddress | where count_ > 1 // Splunk - DCSync Detection index=windows EventCode=4662 | search Properties="*Replicating Directory Changes*" | where NOT match(SubjectUserName, ".*\$$") | table _time, SubjectUserName, SubjectDomainName, ObjectName // KQL - Suspicious Admin Group Changes SecurityEvent | where EventID in (4728, 4732, 4756) | where TargetAccount has_any ("Domain Admins", "Enterprise Admins", "Administrators", "Schema Admins") | project TimeGenerated, SubjectAccount, TargetAccount, MemberName, Activity // Pass-the-Hash Detection (4624 Type 9) index=windows EventCode=4624 LogonType=9 | where NOT match(TargetUserName, ".*\$$") | stats count by TargetUserName, IpAddress, WorkstationName

Azure AD Security

Azure AD Threat Detection

// Suspicious sign-in activity SigninLogs | where RiskLevelDuringSignIn in ("high", "medium") | project TimeGenerated, UserPrincipalName, IPAddress, Location, RiskLevelDuringSignIn, RiskDetail, AppDisplayName, ClientAppUsed // MFA fatigue attack (multiple MFA prompts) SigninLogs | where ResultType == 50074 // MFA required | summarize MFAPrompts = count() by UserPrincipalName, bin(TimeGenerated, 1h) | where MFAPrompts > 10 // Consent phishing detection AuditLogs | where OperationName == "Consent to application" | extend AppName = TargetResources[0].displayName | extend Permissions = AdditionalDetails | project TimeGenerated, InitiatedBy, AppName, Permissions // Privileged role assignment AuditLogs | where OperationName has "Add member to role" | extend Role = TargetResources[0].displayName | where Role has_any ("Global Administrator", "Privileged Role Administrator", "Application Administrator", "Cloud Application Administrator") | project TimeGenerated, InitiatedBy, Role, TargetResources // Suspicious app registrations AuditLogs | where OperationName == "Add application" | extend AppName = TargetResources[0].displayName | project TimeGenerated, InitiatedBy, AppName

Identity Protection Risks

Azure AD Identity Protection Risk Types: USER RISKS (accumulated): - Leaked credentials - Azure AD threat intelligence - Anomalous user activity SIGN-IN RISKS (real-time): - Anonymous IP address - Atypical travel - Unfamiliar sign-in properties - Malware linked IP - Password spray - Suspicious inbox rules Risk Levels: - High: Strong indicator of compromise - Medium: Suspicious activity detected - Low: Minor anomaly - None: No risk detected Response by Risk Level: High → Block + force password reset Medium → Require MFA + review Low → Monitor + investigate if pattern

Credential Theft Detection

LSASS Access Detection

// Sysmon Event 10 - LSASS Access index=sysmon EventCode=10 TargetImage="*\\lsass.exe" | where NOT match(SourceImage, ".*(csrss|lsass|services|svchost|wininit).*") | where GrantedAccess IN ("0x1010", "0x1410", "0x1438", "0x143a") | table _time, Computer, SourceImage, SourceUser, GrantedAccess // KQL - LSASS Access DeviceProcessEvents | where FileName =~ "lsass.exe" | where InitiatingProcessFileName !in~ ("csrss.exe", "services.exe", "svchost.exe", "wininit.exe") | project Timestamp, DeviceName, InitiatingProcessFileName, InitiatingProcessCommandLine Suspicious LSASS Access Patterns: - procdump.exe accessing LSASS - Unknown process with debug privileges - PowerShell accessing LSASS - mimikatz patterns

Credential Dumping Tools

Tool Detection Patterns: MIMIKATZ: - Process name: mimikatz.exe - Command: sekurlsa::logonpasswords - LSASS access with 0x1010 PROCDUMP: - procdump -ma lsass.exe - Creates lsass.dmp file COMSVCS.DLL: - rundll32 comsvcs.dll MiniDump - Creates memory dump NTDSUTIL: - ntdsutil "ac i ntds" "ifm" "create full" - Dumps AD database SECRETSDUMP: - Impacket tool - Remote credential extraction

Privilege Escalation Detection

Privilege Escalation Indicators: GROUP MEMBERSHIP CHANGES: - User added to Domain Admins - User added to local Administrators - Service account privilege increase TOKEN MANIPULATION: - Token impersonation - Privilege escalation via token SCHEDULED TASKS: - Task created to run as SYSTEM - Task with elevated privileges SERVICE MANIPULATION: - Service created with SYSTEM context - Service binary path modification Detection Queries: // KQL - Admin group changes SecurityEvent | where EventID == 4728 | where TargetAccount has "Admins" | project TimeGenerated, SubjectAccount, MemberName, TargetAccount // Splunk - SYSTEM shell index=windows EventCode=4688 | where Creator_Process_Name="*\\services.exe" | where New_Process_Name IN ("*\\cmd.exe", "*\\powershell.exe") | table _time, Computer, New_Process_Name, Process_Command_Line

Identity Incident Response

Compromised Account Response: IMMEDIATE: □ Disable account □ Revoke all sessions (Azure AD/on-prem) □ Reset password □ Require MFA re-registration INVESTIGATION: □ Review sign-in logs □ Check for mailbox rules □ Review accessed resources □ Check for persistence CONTAINMENT: □ Block source IPs □ Disable compromised apps/tokens □ Review group memberships □ Check for lateral movement REMEDIATION: □ Remove unauthorized access □ Scan affected systems □ Reset related credentials □ Update security policies Azure AD Specific: - Sign-in logs: Check locations, apps, devices - Audit logs: Check for changes made - Conditional Access: Block suspicious access - Identity Protection: Review risk events

Interview Questions

  • Pattern: Few passwords tried against many accounts
  • Event 4625: Failed logons from same source
  • Time window: Many unique users in short period
  • Threshold: >10-20 unique accounts from one IP

Key query: Group by source IP, count distinct usernames in 5-minute windows.

  • Attack: Request service tickets for service accounts, crack offline
  • Event 4769: Service ticket requested
  • Indicator: RC4 encryption (0x17) for service accounts
  • Pattern: Multiple service tickets from one user

Detection: Filter 4769 where ServiceName doesn't end with $ and TicketEncryptionType is RC4.

  1. Disable immediately: Both AD and Azure AD
  2. Reset Kerberos tickets: krbtgt if Domain Admin
  3. Review all activity: What was accessed/changed?
  4. Check for persistence: New accounts, scheduled tasks
  5. Reset related passwords: Service accounts they accessed
  6. Scan systems: Where did they authenticate?
