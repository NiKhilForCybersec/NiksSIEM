PCI-DSS 4.0 Requirements

Requirement 10: Log and Monitor All Access

"Track and monitor all access to network resources and cardholder data"

Requirement Description Required Log Sources SIEM Implementation
10.2.1 Individual user access to cardholder data Database audit logs, Application logs, File access logs • Enable database auditing (SQL Server, Oracle, MySQL)
• Monitor SELECT queries on CHD tables
• Alert on bulk data access
10.2.2 Actions by individuals with root/admin privileges Windows Security (4672, 4688), Linux sudo, PAM • Monitor privileged account usage
• Alert on off-hours admin activity
• Track sudo/runas commands
10.2.3 Access to all audit trails SIEM audit logs, Security tool logs • Monitor SIEM admin access
• Alert on audit log deletion attempts
• Track log configuration changes
10.2.4 Invalid logical access attempts Authentication logs (all systems in CDE) • Correlation rule for failed logins
• Brute force detection
• Account lockout monitoring
10.2.5 Use of identification & authentication mechanisms AD logs, RADIUS, TACACS+, MFA logs • Monitor authentication success/failure
• Track MFA bypasses
• Service account usage
10.2.6 Initialization, stopping, pausing of audit logs System logs, Security logs • Alert on audit service stop
• Monitor log agent health
• Detect log clearing events
10.2.7 Creation and deletion of system-level objects OS logs, AD logs • Monitor user/group changes
• Track GPO modifications
• Service installation alerts

PCI-DSS Log Content Requirements (10.3)

Each log entry must contain:

  • 10.3.1: User identification
  • 10.3.2: Type of event
  • 10.3.3: Date and time
  • 10.3.4: Success/failure indication
  • 10.3.5: Origination of event
  • 10.3.6: Identity or name of affected data/resource

HIPAA Security Rule

§164.312 Technical Safeguards

Safeguard Requirement Required Log Sources SIEM Implementation
§164.312(b)
Audit Controls		 REQUIRED EHR/EMR systems, Database, ePHI storage • Collect audit logs from all ePHI systems
• Monitor access patterns
• Detect bulk record access
§164.312(d)
Person/Entity Auth		 REQUIRED Authentication systems, SSO, MFA • Monitor all authentications
• Track session activity
• Detect credential sharing
§164.308(a)(5)
Security Awareness		 REQUIRED Email gateway, Web proxy • Phishing attempt detection
• Malicious link clicks
• Suspicious attachment alerts
§164.308(a)(6)
Incident Procedures		 REQUIRED SIEM alerts, Security tools • Incident detection rules
• Automated alerting
• Incident tracking integration
§164.312(c)(1)
Integrity Controls		 REQUIRED FIM logs, Database audit • File integrity monitoring on ePHI
• Database change detection
• Configuration change alerts

SOC 2 Trust Service Criteria

Common Criteria (CC) Relevant to SIEM

Criteria Requirement SIEM Controls
CC6.1 Logical access security software • Monitor authentication systems
• Track access provisioning/deprovisioning
• Detect unauthorized access attempts
CC6.2 Prior to issuing credentials, register and authorize • Alert on new account creation
• Monitor privilege escalation
• Track access request approvals
CC6.3 Remove access when no longer needed • Correlate HR terminations with access removal
• Alert on orphaned accounts
• Detect access after departure
CC7.2 Monitor system components for anomalies • Security event monitoring
• Anomaly detection rules
• Performance baseline deviation
CC7.3 Evaluate security events • SOC triage process
• Incident classification
• Documented investigation workflow
CC7.4 Respond to security incidents • Incident response procedures
• SOAR automation
• Communication templates
