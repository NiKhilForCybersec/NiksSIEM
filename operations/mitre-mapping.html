MITRE ATT&CK Mapping

Map detections and incidents to the MITRE ATT&CK framework. Understand tactics, techniques, and procedures (TTPs) for better threat intelligence and coverage analysis.

ATT&CK Matrix Overview

What is MITRE ATT&CK?

MITRE ATT&CK is a knowledge base of adversary tactics and techniques based on real-world observations. It provides a common language for describing attacker behavior and helps security teams understand threat coverage.

ATT&CK Structure

LevelDescriptionExample
TacticThe "why" - adversary's goalInitial Access, Persistence, Exfiltration
TechniqueThe "how" - method to achieve goalT1566 Phishing, T1059 Command Line
Sub-techniqueSpecific variationT1566.001 Spearphishing Attachment
ProcedureSpecific implementation by threat actorAPT29 uses spearphishing with LNK files

Coverage Analysis

Using ATT&CK for Coverage

Map your detection rules to ATT&CK techniques to identify gaps:

  • Export rules: List all detection rules with technique tags
  • Map to matrix: Use ATT&CK Navigator to visualize coverage
  • Identify gaps: Which techniques have no detection?
  • Prioritize: Focus on techniques used by threats relevant to you
  • Track progress: Update coverage map as new rules are added

Interview Questions

  • Alert triage: Map alerts to techniques to understand attack stage
  • Investigation: Use technique info to know what else to look for
  • Detection engineering: Write rules targeting specific techniques
  • Threat hunting: Hunt for techniques not well-covered by alerts
  • Reporting: Communicate threats in standard terminology
  • Gap analysis: Identify detection coverage gaps

Tactic: The adversary's goal or objective - the "why"

  • Example: "Persistence" - attacker wants to maintain access

Technique: The method used to achieve the goal - the "how"

  • Example: "Scheduled Task" (T1053) - one way to achieve persistence

One tactic has many techniques. Adversaries choose techniques based on environment and capabilities.
