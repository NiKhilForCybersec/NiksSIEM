SOC Metrics & KPIs

Measure SOC performance with meaningful metrics. Track Mean Time to Detect (MTTD), Mean Time to Respond (MTTR), and other key performance indicators.

Key Metrics

MetricDescriptionTarget
MTTDMean Time to Detect - time from occurrence to detection<24 hours
MTTRMean Time to Respond - time from detection to containment<4 hours
MTTCMean Time to Contain - time to stop active threat<1 hour
FP RateFalse Positive Rate - % of alerts that are FP<30%
Alert VolumeTotal alerts per day/weekVaries
IncidentsConfirmed incidents per monthTrack trend

Operational Metrics

Queue Depth

Alerts waiting for review. High depth indicates staffing or tuning issues.

Time to Triage

How long alerts wait before first review. Track by severity.

Escalation Rate

% of alerts escalated to Tier 2+. Helps with training and staffing.

Recurrence Rate

Same incident type happening repeatedly. Indicates root cause issues.

Detection Metrics

Measuring Detection Capability

  • Detection Coverage: % of ATT&CK techniques with detection rules
  • Rule Efficacy: TP rate per detection rule
  • Detection Sources: Which tools generate most true positives?
  • Blind Spots: Incidents found by external parties, not SOC
  • Purple Team Results: Detection rate during exercises

Interview Questions

Core metrics:

  • MTTD/MTTR: Speed of detection and response
  • False Positive Rate: Alert quality and tuning effectiveness
  • Incidents per month: Workload and threat landscape
  • Detection coverage: % of threats we can detect

Important caveat: Metrics should drive improvement, not gaming. A low MTTR is meaningless if incidents aren't being properly investigated.
