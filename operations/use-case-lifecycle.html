Interview Preparation

How do you develop a new detection use case from scratch?

I follow a 7-phase lifecycle:

1. Ideation: Ideas come from threat intel, past incidents that weren't detected, MITRE ATT&CK gaps, compliance requirements, or red team findings. I prioritize based on threat likelihood vs business impact - high threat + high impact = do now.

2. Development: First, understand the attack - what does it look like in logs? Map to MITRE ATT&CK. Then verify we have the data sources. Write the detection query with: data source, time filter, core logic, exclusions for known benign, thresholds, and output fields for analyst context.

3. Testing: Four test types - Functional (simulate attack, does it detect?), False Positive (run against historical data, is FP rate acceptable?), False Negative (would it have caught past incidents?), and Performance (does query run fast enough?).

4. Deploy: Start in "alert-only" mode for 1-2 weeks. Monitor FP rate. Tune thresholds. Then enable full alerting.

5. Operate: SOC uses it daily. Track effectiveness metrics.

6. Review: Quarterly review - is it still relevant? FP rate acceptable? Needs tuning?

7. Retire: If threat no longer relevant or replaced by better detection.
