Phase 1: Discovery & Planning

Requirements Gathering Checklist

☐ BUSINESS REQUIREMENTS
  ├── ☐ Primary use cases (threat detection, compliance, forensics)
  ├── ☐ Compliance frameworks applicable (PCI, HIPAA, SOC2)
  ├── ☐ Retention requirements by data type
  ├── ☐ SLA expectations (response times, availability)
  └── ☐ Budget constraints
☐ TECHNICAL REQUIREMENTS  
  ├── ☐ Current infrastructure inventory
  ├── ☐ Critical asset list (crown jewels)
  ├── ☐ Data source inventory (see Data Source Priority page)
  ├── ☐ Expected data volume (GB/day)
  ├── ☐ Network architecture (on-prem, cloud, hybrid)
  ├── ☐ Existing security tools to integrate
  └── ☐ User count and roles
☐ OPERATIONAL REQUIREMENTS
  ├── ☐ SOC team size and structure
  ├── ☐ 24x7 coverage needs
  ├── ☐ Existing processes and workflows
  ├── ☐ Training requirements
  └── ☐ Integration with ticketing (ServiceNow, Jira)
☐ STAKEHOLDER SIGN-OFF
  ├── ☐ CISO/Security leadership
  ├── ☐ IT Operations
  ├── ☐ Compliance/Legal
  └── ☐ Finance (budget approval)

Phase 2: Architecture & Setup

Deployment Decisions

Decision Options Considerations
Deployment Model On-prem, Cloud, Hybrid Data residency, cost, scalability, expertise
Sizing Based on GB/day ingestion Current + 50% growth buffer
High Availability Single site, DR, Active-Active RTO/RPO requirements, budget
Retention Tiers Hot, Warm, Cold, Archive Compliance needs, cost optimization

Infrastructure Checklist

☐ COMPUTE & STORAGE
  ├── ☐ Servers/VMs provisioned per sizing guide
  ├── ☐ Storage allocated (hot/warm/cold/archive)
  ├── ☐ Network connectivity configured
  └── ☐ Load balancers (if clustered)
☐ SECURITY
  ├── ☐ TLS certificates for encryption
  ├── ☐ Service accounts created (least privilege)
  ├── ☐ Firewall rules for data ingestion
  ├── ☐ RBAC roles defined
  └── ☐ Integration with identity provider (SAML/OIDC)
☐ PLATFORM INSTALLATION
  ├── ☐ Core SIEM platform installed
  ├── ☐ Cluster configuration (if applicable)
  ├── ☐ License applied and validated
  ├── ☐ Time synchronization (NTP)
  └── ☐ Backup configured

Phase 3: Data Onboarding

Data Onboarding Priority (Waves)

WAVE 1 (Week 1-2): FOUNDATION
─────────────────────────────
Priority: Critical visibility, quick wins
• Active Directory / Identity Provider
• Firewall/VPN (perimeter)
• DNS logs
• Email gateway (O365/Google)
Volume: ~30% of total
WAVE 2 (Week 3-4): ENDPOINTS & NETWORK
──────────────────────────────────────
Priority: Endpoint visibility, network flow
• EDR/AV (MDE, CrowdStrike)
• Windows Security Events
• Linux auth logs
• Proxy/Web gateway
Volume: ~40% of total
WAVE 3 (Week 5): APPLICATIONS & CLOUD
─────────────────────────────────────
Priority: Application layer, cloud
• Cloud audit logs (Azure/AWS/GCP)
• Critical application logs
• Database audit logs
• SaaS security logs (Okta, Salesforce)
Volume: ~20% of total
WAVE 4 (Ongoing): ADDITIONAL SOURCES
────────────────────────────────────
Priority: Nice-to-have, compliance
• Network devices (switches, routers)
• Physical security (badge access)
• DLP logs
• Vulnerability scanner results
Volume: ~10% of total

Phase 4: Operationalize

Go-Live Checklist

☐ DETECTION CONTENT
  ├── ☐ Initial use cases deployed (minimum 20-30)
  ├── ☐ Tuning completed (FP rate <30%)
  ├── ☐ Alert routing configured
  └── ☐ MITRE mapping documented
☐ OPERATIONAL READINESS
  ├── ☐ SOC procedures documented
  ├── ☐ Triage runbooks created
  ├── ☐ Escalation matrix defined
  ├── ☐ Shift schedule set
  └── ☐ On-call rotation established
☐ INTEGRATIONS
  ├── ☐ Ticketing system connected
  ├── ☐ SOAR playbooks (if applicable)
  ├── ☐ Email/Slack alerting
  └── ☐ Threat intelligence feeds
☐ TRAINING
  ├── ☐ Analyst training completed
  ├── ☐ Query language proficiency verified
  ├── ☐ Triage workflow practiced
  └── ☐ Escalation drill conducted
☐ DOCUMENTATION
  ├── ☐ Architecture diagram
  ├── ☐ Data source inventory
  ├── ☐ Use case catalog
  ├── ☐ Admin runbooks
  └── ☐ Disaster recovery plan
☐ GO-LIVE SIGN-OFF
  ├── ☐ SOC Manager approval
  ├── ☐ Security leadership approval
  └── ☐ Hypercare support plan (2-4 weeks)
