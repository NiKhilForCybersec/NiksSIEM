Operational Metrics (Time-Based)

MTTA - Mean Time to Acknowledge

Time from alert creation to first analyst response/assignment

Severity Target Acceptable Breach
Critical < 5 min < 15 min > 15 min
High < 15 min < 30 min > 30 min
Medium < 1 hour < 2 hours > 2 hours
Low < 4 hours < 8 hours > 8 hours

Calculation Query (Splunk)

`notable`
| eval acknowledge_time = if(isnotnull(status_end) AND status="1", status_end - _time, null())
| stats avg(acknowledge_time) as avg_mtta_sec by urgency
| eval avg_mtta_min = round(avg_mtta_sec/60, 2)

Calculation Query (Sentinel)

SecurityIncident
| where Status != "New"
| extend AcknowledgeTime = datetime_diff('minute', FirstModifiedTime, CreatedTime)
| summarize AvgMTTA = avg(AcknowledgeTime) by Severity

MTTR - Mean Time to Resolve

Time from alert creation to incident closure

Severity Target Acceptable Breach
Critical < 4 hours < 8 hours > 8 hours
High < 8 hours < 24 hours > 24 hours
Medium < 24 hours < 72 hours > 72 hours
Low < 72 hours < 1 week > 1 week

False Positive Rate

Percentage of alerts that are not actual security incidents

┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                        FALSE POSITIVE RATE CALCULATION                                   │
├─────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                          │
│                     FP Rate = (False Positives / Total Alerts) × 100                    │
│                                                                                          │
│   ┌─────────────────────────────────────────────────────────────────────────────────┐   │
│   │  Example:                                                                        │   │
│   │                                                                                  │   │
│   │  Total Alerts This Month:     1,000                                             │   │
│   │  True Positives (TP):           200  (actual security events)                   │   │
│   │  False Positives (FP):          700  (benign activity)                          │   │
│   │  Inconclusive:                  100  (insufficient evidence)                    │   │
│   │                                                                                  │   │
│   │  FP Rate = 700 / 1000 × 100 = 70%  ← NEEDS TUNING                               │   │
│   │                                                                                  │   │
│   └─────────────────────────────────────────────────────────────────────────────────┘   │
│                                                                                          │
│   TARGET BENCHMARKS:                                                                     │
│   ─────────────────                                                                      │
│   • Excellent:  < 10% FP rate                                                           │
│   • Good:       10-20% FP rate                                                          │
│   • Acceptable: 20-30% FP rate                                                          │
│   • Needs Work: 30-50% FP rate                                                          │
│   • Critical:   > 50% FP rate (analyst fatigue risk)                                    │
│                                                                                          │
│   TRACK BY RULE:                                                                         │
│   ──────────────                                                                         │
│   Identify rules with highest FP rates for prioritized tuning                           │
│                                                                                          │

Coverage Metrics

Asset Coverage

Percentage of critical assets sending logs to SIEM

Asset Coverage = (Assets in SIEM / Total Critical Assets) × 100
Categories to measure:
• Servers (Windows, Linux)           Target: 100%
• Network devices (FW, switches)     Target: 100%
• Endpoints (workstations)           Target: 95%
• Cloud resources                    Target: 100%
• Applications (critical apps)       Target: 100%
• Identity systems                   Target: 100%

Interview Preparation

What KPIs do you use to measure SOC effectiveness?

I track metrics across three categories:

Operational Efficiency:

  • MTTA (Mean Time to Acknowledge): How fast we start working on alerts. Target: <15 min for critical.
  • MTTR (Mean Time to Resolve): End-to-end resolution time. Target: <4 hours for critical.
  • False Positive Rate: % of alerts that aren't real threats. Target: <30%. High FP means wasted analyst time.
  • Backlog: Unworked alerts. Should stay below 50 at any time.

Detection Effectiveness:

  • True Positive Rate: % of alerts that are real incidents.
  • Incident Escalation Rate: % that become actual incidents requiring response.
  • Detection Coverage: MITRE ATT&CK coverage percentage.

Coverage:

  • Asset Coverage: % of critical assets logging to SIEM (target: 100%).
  • Log Source Health: % of expected logs actually arriving.

I report these weekly to SOC leadership, monthly to executives, with trend analysis to show improvement over time.
