Response Actions

Master incident response capabilities in Microsoft Defender for Endpoint. Learn to contain threats, investigate devices, and remediate security incidents effectively.

Response Actions Overview

Security.microsoft.com → Devices → [Device] → Response Actions

Available Response Actions

ActionDescriptionUse Case
Isolate Device Block network except MDE Active compromise
Restrict App Execution Allow only Microsoft-signed Malware containment
Run Antivirus Scan Trigger quick or full scan Malware detection
Collect Investigation Package Gather forensic data Deep investigation
Initiate Live Response Remote shell session Manual investigation
Initiate Automated Investigation Run AIR playbooks Automated remediation

Device Isolation

Isolation Types

Full Isolation: - Blocks ALL network connections - Only MDE cloud communication allowed - User sees network disconnected - Most restrictive option Selective Isolation: - Allows Outlook, Teams, Skype - Blocks other network traffic - Better user experience - Still contains most threats

When to Isolate

✅ Isolate When:
  • Active malware or ransomware detected
  • Confirmed C2 communication
  • Credential theft in progress
  • Lateral movement observed
  • Data exfiltration suspected

Isolation Workflow

1. Navigate to Device Page Security.microsoft.com → Devices → [Device Name] 2. Click "Isolate Device" - Choose Full or Selective - Add comment (required) - Confirm action 3. Verify Isolation - Check device status - "Isolated" badge appears - Network indicator shows isolated 4. Continue Investigation - Use Live Response (still works) - Collect investigation package - Review timeline 5. Release Isolation - "Release from Isolation" when safe - Verify remediation complete - Monitor for recurrence

Live Response

Device → Initiate Live Response Session

Live Response Capabilities

Session Types: Basic Session: - Read-only commands - File browsing - Process listing - No modifications Advanced Session: - Full command execution - File upload/download - Script execution - Remediation actions

Basic Commands

# Navigation cd [path] # Change directory dir # List directory contents # Process Information processes # List running processes connections # Show network connections # File Operations fileinfo [file] # Get file details getfile [file] # Download file for analysis # System Information registry # Query registry (with subcommands) trace # Capture ETW trace # Help help # Show available commands help [command] # Command-specific help

Advanced Commands

# Process Management remediate file [path] # Quarantine file remediate process [pid] # Kill and quarantine # File Operations putfile [localfile] # Upload file to device run [script] # Execute uploaded script # Cleanup undo file [path] # Restore quarantined file undo process [path] # Restore and run process # Library Scripts library # List available scripts run [script] [args] # Run library script

Live Response Examples

# Investigate suspicious process processes | find "powershell" connections | find "powershell" # Get suspicious file for analysis cd C:\Users\victim\AppData\Local\Temp dir fileinfo malware.exe getfile malware.exe # Kill malicious process processes # Note the PID remediate process 1234 # Quarantine malicious file remediate file C:\Users\victim\Downloads\malware.exe # Upload and run investigation script putfile collect_artifacts.ps1 run collect_artifacts.ps1

Investigation Package

Package Contents

Investigation Package Includes: Autoruns: - Startup programs - Scheduled tasks - Services - Browser extensions Installed Programs: - Software inventory - Version information Network: - Network configuration - Active connections - DNS cache - ARP cache Prefetch Files: - Program execution history - Timestamps Processes: - Running processes - Loaded modules - Command lines Security Event Log: - Recent security events - Authentication logs Services: - Service configurations - Service status Windows Firewall: - Firewall rules - Connection logs

Using Investigation Package

1. Collect Package Device → Collect investigation package Wait for collection (can take several minutes) 2. Download Package Action center → Download when ready ZIP file downloaded 3. Extract and Analyze - Review autoruns for persistence - Check processes for malware - Analyze network connections - Correlate with timeline 4. Key Files to Review - Autoruns.csv - Persistence mechanisms - Network_*.txt - C2 connections - Processes_*.txt - Running malware - Prefetch - Execution evidence

AIR Actions

Automated Actions Include: File Actions: - Quarantine malicious files - Block file hash - Remove persistence Process Actions: - Stop malicious processes - Remove from memory Entity Actions: - Mark safe (false positive) - Confirm threat Investigation Actions: - Expand investigation scope - Check related devices - Analyze related files

Indicators of Compromise

Settings → Endpoints → Indicators

Indicator Types

TypeFormatActions

IP Address IPv4, IPv6 Allow, Audit, Block
URL/Domain Full URL or domain Allow, Audit, Block
Certificate Thumbprint Allow, Block

Creating Indicators

Indicator Configuration: 1. Navigate to Indicators Settings → Endpoints → Indicators 2. Add Indicator - Select type (File, IP, URL, Certificate) - Enter value - Choose action - Set scope (all devices or groups) - Add title and description 3. Indicator Actions Allow: Whitelist, prevent detection Audit: Log but don't block Block: Prevent execution/connection Block and Remediate: Block + auto-remove 4. Best Practices - Document reason for indicator - Set expiration for temporary blocks - Review indicators regularly - Use threat intelligence feeds

Attack Surface Reduction

Settings → Endpoints → Attack Surface Reduction

ASR Rules

Key ASR Rules: Office Rules: - Block Office from creating executable content - Block Office from creating child processes - Block Office from injecting into processes - Block Win32 API calls from Office macros Script Rules: - Block obfuscated scripts - Block JavaScript/VBScript launching downloads - Block execution of potentially obfuscated scripts Email Rules: - Block executable content from email - Block executable files from web mail Credential Rules: - Block credential stealing from LSASS - Block process creations from PSExec/WMI Ransomware: - Use advanced protection against ransomware Exploit Rules: - Block abuse of exploited vulnerable signed drivers

ASR Modes

ModeValueBehavior
Disabled0Rule not active
Block1Block and log
Audit2Log only
Warn6Warn user, allow bypass

Response Playbooks

Malware Response

Malware Response Playbook: 1. CONTAIN □ Isolate device (Full isolation) □ Block file hash □ Identify affected scope 2. INVESTIGATE □ Review device timeline □ Check process tree □ Identify entry point □ Look for persistence □ Check for lateral movement 3. ERADICATE □ Run antivirus scan □ Use Live Response to remove artifacts □ Clear persistence mechanisms □ Remove scheduled tasks 4. RECOVER □ Verify malware removed □ Release from isolation □ Monitor for recurrence □ Update detections

Credential Theft Response

Credential Theft Response: 1. IMMEDIATE ACTIONS □ Isolate affected device □ Reset compromised passwords □ Revoke active sessions 2. SCOPE ASSESSMENT □ Identify accessed accounts □ Check for lateral movement □ Review authentication logs 3. INVESTIGATION □ Collect investigation package □ Analyze LSASS access □ Check for credential dumping tools □ Review network connections 4. REMEDIATION □ Reset all potentially compromised creds □ Enable MFA if not present □ Update to credential guard □ Monitor for abuse

Interview Questions

Isolate when:

  • Active C2 communication detected
  • Ransomware or destructive malware
  • Credential theft in progress
  • Lateral movement observed

Consider alternatives when:

  • Low severity, automated remediation sufficient
  • Business-critical system (coordinate first)
  • Detection may be false positive
  • Investigation: Browse files, list processes, check connections
  • Collection: Download files, capture memory, get registry
  • Remediation: Kill processes, quarantine files, run scripts
  • Forensics: Collect artifacts, capture ETW traces

Live Response works even on isolated devices, enabling continued investigation during containment.

  • Audit mode: Logs events but doesn't block - use for testing impact before enforcement
  • Block mode: Prevents action and logs - full protection

Best practice: Deploy rules in audit mode first, analyze logs for false positives, tune exclusions, then enable block mode.
