KQL Intermediate

Build on KQL fundamentals with joins, unions, let statements, and advanced aggregations. These techniques are essential for complex threat hunting and detection engineering.

Let Statements

Use let to create reusable variables, tables, and functions within a query.

Let Statement Examples
// Simple variable
let threshold = 10;
let timeframe = 24h;

// Table expression
let FailedLogins = SigninLogs
| where ResultType != 0
| where TimeGenerated > ago(timeframe);

// Use in query
FailedLogins
| summarize count() by UserPrincipalName
| where count_ > threshold

// Function
let GetFailures = (user:string) {
    SigninLogs
    | where UserPrincipalName == user
    | where ResultType != 0
};
GetFailures("admin@contoso.com")

Join Operations

Join Types
// Inner join (default) - matching rows only
SigninLogs
| join kind=inner (
    IdentityInfo | project UserPrincipalName, Department
) on UserPrincipalName

// Left outer - all from left, matching from right
SigninLogs
| join kind=leftouter (
    ThreatIntelligence | project IPAddress, ThreatType
) on $left.IPAddress == $right.IPAddress

// Anti join - rows in left NOT in right
SigninLogs
| join kind=leftanti (
    AllowedIPList | project IPAddress
) on IPAddress
// Returns sign-ins from IPs NOT in allow list

// Semi join - rows in left that HAVE matches
SigninLogs
| join kind=leftsemi (
    WatchlistUsers | project UserPrincipalName
) on UserPrincipalName
// Returns sign-ins only for watchlist users

Union Operations

Union Examples
// Combine tables with same schema
SecurityEvent
| where EventID == 4625
| union (
    SigninLogs
    | where ResultType != 0
)
| summarize count() by bin(TimeGenerated, 1h)

// Union with wildcard (all tables matching pattern)
union Security*
| where TimeGenerated > ago(1h)
| summarize count() by Type

// Union with different schemas
SecurityEvent | project TimeGenerated, Account, Computer, Activity
| union (
    SigninLogs | project TimeGenerated, Account=UserPrincipalName, Computer="Azure", Activity="Signin"
)

Interview Questions

inner: Returns only rows that match in both tables

leftouter: Returns all rows from left table, matching data from right (null if no match)

leftanti: Returns rows from left table that have NO match in right table

Use cases:

  • inner: Correlate events with threat intel
  • leftouter: Enrich logs with additional context
  • leftanti: Find events NOT in whitelist
