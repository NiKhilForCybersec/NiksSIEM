Interview Preparation

What are the main methods for ingesting data into Sentinel?

There are several ingestion methods depending on the source:

Native connectors are one-click integrations in Content Hub - Microsoft services like Entra ID, M365, Defender use these. API-based with no infrastructure needed.

Azure Monitor Agent (AMA) with Data Collection Rules is the modern approach for Windows/Linux servers. Install AMA, create DCRs that specify what events to collect and where to send them. This replaced the legacy Log Analytics agent.

Syslog/CEF for network devices. You set up a Linux VM as a forwarder, devices send syslog to it, AMA forwards to Sentinel. CEF-formatted logs get parsed automatically into the CommonSecurityLog table.

Logs Ingestion API for custom sources. You POST data to a Data Collection Endpoint using DCR for transformation. Good for custom apps or sources without native connectors.

Azure Functions for SaaS with REST APIs - the function polls the external API on a schedule and sends data to Sentinel. Many Content Hub solutions use this pattern.
