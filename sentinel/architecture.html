Sentinel Architecture

Understanding Microsoft Sentinel's architecture is crucial for designing scalable, cost-effective security operations. This guide covers workspace design, multi-tenant patterns, regional considerations, and best practices.

Architecture Foundation

Sentinel is built on Azure Log Analytics workspaces. Every Sentinel deployment starts with a Log Analytics workspace - this is where all your data lives, queries run, and analytics rules execute.

Core Components

Log Analytics Workspace

Foundation of Sentinel. Stores all ingested data in tables, runs KQL queries, manages retention. One workspace per Sentinel instance.

Data Connectors

200+ native connectors for Microsoft, cloud, and 3rd party sources. Handles authentication, data transformation, and ingestion.

Analytics Engine

Runs scheduled rules, NRT rules, Fusion ML, and anomaly detection. Generates alerts that become incidents.

Automation Layer

Logic Apps integration for playbooks. Triggers on alerts/incidents for enrichment, containment, notification.

Workspace Design Patterns

Single Workspace (Recommended for Most)

When to Use Single Workspace

  • Single organization with unified security team
  • No strict data residency requirements
  • Simpler management and correlation
  • Cost optimization through commitment tiers

Multiple Workspaces

When Multiple Workspaces are Needed

  • Data residency: EU data must stay in EU region
  • MSSP/Multi-tenant: Customer data isolation
  • Access control: Strict separation between business units
  • Cost allocation: Separate billing per department

Trade-off: Cross-workspace queries are possible but slower. Correlation across workspaces requires additional configuration.

Interview Questions

Answer: For multinational with EU data residency:

  • Multiple workspaces: One in EU region, one in US/other region
  • Data routing: DCRs route EU-origin data to EU workspace
  • Cross-workspace queries: Use workspace() function for unified hunting
  • Azure Lighthouse: For centralized SOC view across workspaces
  • Shared analytics rules: Deploy same rules to both via ARM templates

Single workspace benefits: Easier correlation, simpler management, better cost optimization

Multiple workspace triggers:

  • Regulatory data residency requirements
  • MSSP with customer isolation needs
  • Strict RBAC between business units
  • Separate cost centers requiring distinct billing
