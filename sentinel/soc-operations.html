Quick Investigation Queries

Common Investigation KQL
// All activity for a user
let targetUser = "suspicious@company.com";
union SigninLogs, AuditLogs, SecurityAlert
| where TimeGenerated > ago(7d)
| where UserPrincipalName =~ targetUser or TargetUserPrincipalName =~ targetUser
| project TimeGenerated, Type, OperationName, Result = coalesce(ResultType, Result)
| sort by TimeGenerated desc

// All activity from an IP
let targetIP = "1.2.3.4";
union SigninLogs, CommonSecurityLog, AzureActivity
| where TimeGenerated > ago(24h)
| where IPAddress == targetIP or SourceIP == targetIP or CallerIpAddress == targetIP
| project TimeGenerated, Type, Account = coalesce(UserPrincipalName, SourceUserName)

// Device timeline
let targetDevice = "WORKSTATION01";
SecurityEvent
| where TimeGenerated > ago(24h)
| where Computer =~ targetDevice
| project TimeGenerated, EventID, Activity, Account
| sort by TimeGenerated desc

Interview Questions

  1. Review alert details - what triggered it? What entities involved?
  2. Check entity page - user's recent activity, sign-in locations, devices
  3. Look for IOCs - impossible travel, new device, unusual app access
  4. Expand investigation - what did user access after suspicious sign-in?
  5. Check related alerts - any other alerts for same user or from same IP?
  6. Determine scope - was data accessed/exfiltrated? Other users affected?
  7. Containment - if confirmed, disable account, revoke sessions
  8. Document - record findings, timeline, actions taken
