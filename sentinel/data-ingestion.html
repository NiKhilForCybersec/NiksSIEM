Data Ingestion & Cost

Efficient data ingestion is critical for both security coverage and cost management. Learn strategies to ingest the right data at the right cost.

Cost Factors

FactorImpactOptimization
Volume (GB)Primary cost driverFilter unnecessary data at source
Log tierAnalytics vs Basic vs ArchiveUse Basic for verbose, low-value logs
RetentionLonger = more costUse archive tier for compliance
QueriesData scanned for Basic logsTime-bound queries, use Analytics for frequent

Log Tiers Explained

Analytics Logs

Full KQL query capabilities. 30-day interactive + 12-year archive. Default tier. ~$2.76/GB

Basic Logs

Limited queries (8 days, simple ops). 8x cheaper (~$0.50/GB). Good for verbose logs.

Archive Tier

Long-term storage. Must restore to query. Very cheap. Use for compliance/forensics.

Interview Questions

Strategies:

  1. Commitment tiers: Move from PAYG to commitment tier (25-50% savings)
  2. Basic logs: Move verbose logs (debug, informational) to Basic tier
  3. DCR filtering: Drop unnecessary fields and events at ingestion
  4. Review data sources: Disable redundant connectors
  5. Archive old data: Move data >90 days to archive tier
