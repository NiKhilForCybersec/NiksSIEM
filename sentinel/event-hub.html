Event Hub Integration

Azure Event Hubs enable high-volume, real-time data streaming into Sentinel. Use for sources that can't use native connectors or need custom processing.

When to Use Event Hub

High Volume

Sources sending millions of events/second that need buffering

Custom Processing

Need to transform, filter, or enrich before ingestion

Multi-Destination

Same data going to Sentinel + other systems (SIEM, data lake)

No Native Connector

Custom apps or unsupported sources that can write to Event Hub

Interview Questions

Use native connector when:

  • Connector exists for your source
  • Volume is manageable
  • No custom transformation needed

Use Event Hub when:

  • Very high volume requiring buffering
  • Need to process/transform data before ingestion
  • Same data goes to multiple destinations
  • Custom application that can write to Event Hub
