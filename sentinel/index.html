Quick Start KQL Queries

KQL Failed Sign-ins by User 
SigninLogs
| where TimeGenerated > ago(24h)
| where ResultType != "0"
| summarize FailedAttempts = count() by UserPrincipalName, IPAddress
| where FailedAttempts > 5
| sort by FailedAttempts desc
KQL Security Alerts Summary 
SecurityAlert
| where TimeGenerated > ago(7d)
| summarize AlertCount = count() by AlertSeverity, ProductName
| order by AlertSeverity asc, AlertCount desc
| render barchart
KQL Rare Process Execution 
DeviceProcessEvents
| where TimeGenerated > ago(7d)
| summarize ExecutionCount = count() by FileName, FolderPath
| where ExecutionCount < 5
| join kind=inner (
    DeviceProcessEvents
    | where TimeGenerated > ago(1h)
) on FileName
| project TimeGenerated, DeviceName, FileName, FolderPath, ProcessCommandLine

Interview Preparation

What are the advantages of Microsoft Sentinel over traditional SIEM?
Key Points:

Sentinel offers several advantages over traditional on-premises SIEM solutions: First, cloud-native scalability eliminates capacity planning and infrastructure management. Second, consumption-based pricing with commitment tiers provides cost flexibility. Third, native Microsoft integration enables seamless data collection from M365, Azure AD, and Defender products. Fourth, Fusion ML detection automatically correlates alerts across data sources to identify sophisticated attacks. Fifth, built-in SOAR capabilities through Logic Apps integration enables automated response without additional licensing.

How do you optimize Sentinel costs?
Cost Optimization Strategies:

Several techniques reduce Sentinel costs: Commitment tiers offer up to 50% savings for predictable volumes. Basic Logs tier reduces costs by 60% for high-volume, low-query data. Data Collection Rules filter and transform data before ingestion. Archive tier stores long-term data at minimal cost. Workspace design consolidates data to maximize commitment tier benefits while using separate workspaces only when required for compliance or data sovereignty.
