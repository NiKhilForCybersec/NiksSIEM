Threat Intelligence

Complete guide to Microsoft Sentinel Threat Intelligence. Configure feeds, manage indicators, create TI-based analytics, and hunt for IOC matches.

TI Overview

Sentinel Threat Intelligence capabilities:

  • Ingest indicators from TAXII feeds, MISP, Microsoft Defender TI, or manual upload
  • Store in ThreatIntelligenceIndicator table
  • Match against incoming logs with built-in analytics rules
  • Hunt using KQL queries across all data

Supported Indicator Types

IP Address (ipv4/ipv6-addr) Domain (domain-name) File Hash (file) URL (url) Email (email-addr)

ThreatIntelligenceIndicator Table

All indicators stored in ThreatIntelligenceIndicator table:

// View all active indicators ThreatIntelligenceIndicator | where TimeGenerated > ago(30d) | where Active == true | project TimeGenerated, IndicatorType, NetworkIP, DomainName, FileHashValue, Confidence, ThreatType, Source, ExpirationDateTime | take 100 // Count indicators by type and source ThreatIntelligenceIndicator | where Active == true | summarize Count = count() by IndicatorType, Source | order by Count desc // Find expiring indicators ThreatIntelligenceIndicator | where Active == true | where ExpirationDateTime between (now() .. now() + 7d) | project DomainName, NetworkIP, FileHashValue, ExpirationDateTime, Source

Key Fields

FieldDescription
IndicatorTypeType: ipv4-addr, domain-name, file, url, email-addr
NetworkIPIP address indicator
DomainNameDomain indicator
FileHashValueFile hash (MD5, SHA1, SHA256)
UrlURL indicator
Confidence0-100 confidence score
ThreatTypeMalware, C2, Phishing, Botnet, etc.
SourceFeed name
ActiveTrue if not expired/revoked

TI-Based Analytics Rules

Built-in TI Matching Rules

Content hub → Threat Intelligence solution includes:

  • TI map IP entity to AzureActivity
  • TI map IP entity to SigninLogs
  • TI map IP entity to CommonSecurityLog
  • TI map Domain entity to DNS events
  • TI map URL entity to proxy logs
  • TI map FileHash entity to endpoint data

Custom TI Matching Query

// Match IPs against threat intel let TI_IPs = ThreatIntelligenceIndicator | where Active == true and IndicatorType == "ipv4-addr" | project NetworkIP, Confidence, ThreatType, Source; SigninLogs | where TimeGenerated > ago(1d) | join kind=inner TI_IPs on $left.IPAddress == $right.NetworkIP | project TimeGenerated, UserPrincipalName, IPAddress, TI_Confidence = Confidence, TI_ThreatType = ThreatType, TI_Source = Source // Match domains against threat intel let TI_Domains = ThreatIntelligenceIndicator | where Active == true and IndicatorType == "domain-name" | project DomainName, Confidence, ThreatType; DnsEvents | where TimeGenerated > ago(1d) | join kind=inner TI_Domains on $left.Name == $right.DomainName | project TimeGenerated, Computer, ClientIP, QueryName = Name, TI_Confidence = Confidence, TI_ThreatType = ThreatType

Threat Hunting with TI

// Hunt for any IP matches across all data let TI_IPs = ThreatIntelligenceIndicator | where Active == true and IndicatorType == "ipv4-addr" | distinct NetworkIP; search in (SigninLogs, AzureActivity, CommonSecurityLog, SecurityEvent) "*" | where TimeGenerated > ago(7d) | where SourceIP in (TI_IPs) or DestinationIP in (TI_IPs) or IPAddress in (TI_IPs) | summarize Count = count() by Type, SourceIP, DestinationIP // Hunt for file hash matches in MDE let TI_Hashes = ThreatIntelligenceIndicator | where Active == true and IndicatorType == "file" | project FileHashValue; DeviceFileEvents | where TimeGenerated > ago(7d) | where SHA256 in (TI_Hashes) or MD5 in (TI_Hashes) | project TimeGenerated, DeviceName, FileName, FolderPath, SHA256, InitiatingProcessFileName

Interview Questions

  1. Install connector: Content hub → Threat Intelligence - TAXII (or MISP, Microsoft TI)
  2. Configure feed: Data connectors → Enter API root URL, collection ID, credentials
  3. Enable matching rules: Content hub → Threat Intelligence solution → Enable analytics rules
  4. Verify ingestion: Query ThreatIntelligenceIndicator table

Key sources: TAXII 2.x feeds (AlienVault OTX, Anomali), Microsoft Defender TI, MISP, manual CSV upload, Graph API.

Method 1 - Built-in rules: Enable "TI map" analytics rules from Threat Intelligence solution. These auto-match IPs, domains, URLs, hashes against common tables.

Method 2 - Custom KQL:

  1. Query ThreatIntelligenceIndicator for active indicators
  2. Join against target table (SigninLogs, CommonSecurityLog, etc.)
  3. Match on NetworkIP, DomainName, FileHashValue fields

Best practice: Use confidence threshold (e.g., Confidence > 70) to reduce false positives.

Additional Interview Questions

Systematic troubleshooting approach:

  1. Identify the symptom: What exactly is not working as expected?
  2. Check connectivity: Verify network paths and authentication
  3. Review logs: Check internal/audit logs for errors
  4. Validate configuration: Compare against documented standards
  5. Test incrementally: Isolate the problem component
  6. Document findings: Record what worked and what didn't

Key performance indicators:

  • Availability: Uptime percentage, service availability
  • Performance: Query response time, processing latency
  • Volume: Events per second, data ingestion rate
  • Quality: Parse success rate, enrichment coverage
  • Efficiency: Resource utilization, cost per GB

High availability strategies:

  • Redundancy: Multiple instances of critical components
  • Load balancing: Distribute traffic across instances
  • Failover: Automatic switchover on failure
  • Monitoring: Health checks and alerting
  • DR Planning: Documented recovery procedures
