Workbooks

Workbooks are interactive dashboards that combine KQL queries with visualizations. Use them for security monitoring, reporting, and investigation workflows.

Workbook Components

Query Steps

KQL queries that fetch data. Can be parameterized and linked to other steps.

Parameters

Dropdowns, time pickers, text inputs for interactive filtering. Drive query variables.

Visualizations

Charts, grids, tiles, maps. Multiple visualization types per query.

Links & Actions

Navigate to other workbooks, run playbooks, open investigation blades.

Common Workbook Queries

Security Dashboard Queries
// Incident summary tile
SecurityIncident
| where TimeGenerated > ago({TimeRange})  // Parameter
| summarize 
    Total = count(),
    High = countif(Severity == "High"),
    Medium = countif(Severity == "Medium"),
    Low = countif(Severity == "Low")

// Failed logins trend
SigninLogs
| where TimeGenerated > ago({TimeRange})
| where ResultType != 0
| summarize Failures = count() by bin(TimeGenerated, {TimeGrain})  // Parameter
| render timechart

// Top attackers
SigninLogs
| where TimeGenerated > ago({TimeRange})
| where ResultType != 0
| summarize Attempts = count() by IPAddress
| top 10 by Attempts
| render barchart

Interview Questions

Best practices:

  • Use parameters: Time range, workspace, subscription selectors
  • Export as template: Gallery template or ARM template
  • Store in Content Hub: Package with solution for deployment
  • Version control: Export JSON and store in Git
