Content Hub & ASIM

Content Hub provides pre-built solutions, analytics rules, workbooks, and playbooks. ASIM (Advanced Security Information Model) normalizes data across sources for unified detection.

Content Hub

What is Content Hub?

Content Hub is Sentinel's marketplace for packaged security content. Each solution includes data connectors, analytics rules, workbooks, hunting queries, and playbooks - everything you need to monitor a specific product or use case.

ASIM (Advanced Security Information Model)

Why ASIM Matters

Every vendor names fields differently. ASIM provides normalized schemas so one query works across ALL sources. Write detection once, apply everywhere.

ASIM Schemas

SchemaParser FunctionKey Normalized Fields
AuthenticationimAuthenticationTargetUsername, SrcIpAddr, EventResult
DNSimDnsDnsQuery, DnsResponseCode, SrcIpAddr
Network SessionimNetworkSessionSrcIpAddr, DstIpAddr, DstPortNumber
Process EventimProcessCreateTargetProcessName, CommandLine, ActorUsername
Web SessionimWebSessionUrl, HttpStatusCode, SrcIpAddr

Using ASIM Parsers

ASIM Query Examples
// Failed authentication across ALL sources
imAuthentication
| where TimeGenerated > ago(1h)
| where EventResult == "Failure"
| summarize FailCount = count() by TargetUsername, SrcIpAddr
| where FailCount > 10

// DNS queries to rare domains
imDns
| where TimeGenerated > ago(24h)
| summarize QueryCount = count() by DnsQuery
| where QueryCount < 5
| join kind=inner (
    imDns | where TimeGenerated > ago(1h)
) on DnsQuery

// Process creation with suspicious patterns
imProcessCreate
| where TimeGenerated > ago(1h)
| where TargetProcessName has_any ("powershell", "cmd", "wscript")
| where CommandLine contains "-enc" or CommandLine contains "bypass"

Interview Questions

ASIM (Advanced Security Information Model) is Microsoft's data normalization framework:

  • Defines standard schemas for common security event types
  • Provides parser functions that normalize vendor-specific data
  • Enables one query to work across multiple data sources
  • Out-of-box detections automatically work with new sources

Example: Instead of separate queries for Windows, Azure AD, and Okta auth failures, use: imAuthentication | where EventResult == "Failure"
