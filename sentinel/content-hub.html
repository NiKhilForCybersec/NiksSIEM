Content Hub

Discover and deploy out-of-box solutions including data connectors, analytics rules, workbooks, and playbooks from Microsoft and partners.

What is Content Hub?

Content Hub is the marketplace for Sentinel solutions. Each solution is a package containing:

  • Data Connectors: Ingest data from specific sources
  • Analytics Rules: Pre-built detection rules
  • Hunting Queries: Threat hunting starting points
  • Workbooks: Visualization dashboards
  • Playbooks: Automation workflows
  • Parsers: KQL functions for data normalization

Accessing Content Hub

Sentinel → Content hub

Navigation

  • All: Browse all available solutions
  • Installed: View and manage installed solutions
  • Updates: Solutions with available updates
  • Filter by: Category, Provider, Support model, Content type

Installing Solutions

Step-by-Step Installation

  1. Navigate to Content hub
  2. Search for solution (e.g., "Azure Active Directory")
  3. Click on solution card
  4. Review included content
  5. Click Install
  6. After installation, configure data connectors
  7. Enable desired analytics rules

Post-Installation Steps

  • Data Connectors: Open connector page, complete configuration
  • Analytics Rules: Go to Analytics → Rule templates → Enable rules
  • Workbooks: View in Workbooks section
  • Playbooks: Configure in Automation section

Managing Installed Solutions

Content hub → Installed

Update Solutions

  1. Check "Updates" tab for available updates
  2. Click on solution
  3. Click "Update"
  4. Review changelog

Uninstall Solutions

  1. Select installed solution
  2. Click "Manage"
  3. Click "Uninstall"
  4. Note: Data connectors and enabled rules persist

Using Analytics Rule Templates

Sentinel → Analytics → Rule templates

Enable a Rule from Template

  1. Find rule template (filter by solution, data source)
  2. Click "Create rule"
  3. Review and customize settings
  4. Adjust query if needed
  5. Configure entity mapping
  6. Set automated response (optional)
  7. Enable rule

Customizing Templates

// Original template query SigninLogs | where ResultType != 0 | summarize count() by UserPrincipalName // Customized with exclusions let ExcludedUsers = dynamic(["service-account@company.com"]); SigninLogs | where ResultType != 0 | where UserPrincipalName !in (ExcludedUsers) | summarize count() by UserPrincipalName | where count_ > 10 // Add threshold

Best Practices

  • Start with Microsoft solutions: Native integration, free data with E5
  • Enable rules gradually: Don't enable all rules at once - tune as you go
  • Review rule logic: Understand what each rule detects before enabling
  • Customize thresholds: Adjust for your environment to reduce noise
  • Keep solutions updated: Microsoft regularly adds new detections
  • Use rule templates: Faster than writing from scratch

Additional Interview Questions

Systematic troubleshooting approach:

  1. Identify the symptom: What exactly is not working as expected?
  2. Check connectivity: Verify network paths and authentication
  3. Review logs: Check internal/audit logs for errors
  4. Validate configuration: Compare against documented standards
  5. Test incrementally: Isolate the problem component
  6. Document findings: Record what worked and what didn't

Key performance indicators:

  • Availability: Uptime percentage, service availability
  • Performance: Query response time, processing latency
  • Volume: Events per second, data ingestion rate
  • Quality: Parse success rate, enrichment coverage
  • Efficiency: Resource utilization, cost per GB

High availability strategies:

  • Redundancy: Multiple instances of critical components
  • Load balancing: Distribute traffic across instances
  • Failover: Automatic switchover on failure
  • Monitoring: Health checks and alerting
  • DR Planning: Documented recovery procedures

Troubleshooting Guide

Common Issues and Solutions

IssuePossible CauseSolution
Data not appearing Collection or parsing issue Check collector status, verify parser configuration
Slow performance Resource constraints or inefficient queries Review resource utilization, optimize queries
Authentication failures Credential or permission issues Verify credentials, check RBAC configuration
Missing fields Parser not extracting fields Review parsing rules, update field extraction
Duplicate events Multiple collection paths Audit data sources, implement deduplication

Diagnostic Steps

  1. Check Status: Verify component health in console
  2. Review Logs: Examine internal logs for errors
  3. Test Connectivity: Verify network paths and authentication
  4. Validate Config: Compare configuration against documentation
  5. Isolate Issue: Narrow down to specific component
  6. Escalate: Contact support if needed with diagnostics
