DCR & DCE Guide

Data Collection Rules (DCR) and Data Collection Endpoints (DCE) are the modern way to collect and transform data in Azure Monitor and Sentinel. Master these for efficient data ingestion.

Key Concepts

Data Collection Rule (DCR)

Defines WHAT data to collect, HOW to transform it, and WHERE to send it. JSON configuration.

Data Collection Endpoint (DCE)

Network endpoint where agents send data. Required for custom logs and some scenarios.

Azure Monitor Agent (AMA)

Modern agent that uses DCRs. Replaces legacy agents (MMA, OMS).

DCR Structure

DCR JSON Example
{
  "location": "eastus",
  "properties": {
    "dataSources": {
      "windowsEventLogs": [{
        "name": "SecurityEvents",
        "streams": ["Microsoft-SecurityEvent"],
        "xPathQueries": [
          "Security!*[System[(EventID=4624 or EventID=4625)]]"
        ]
      }],
      "syslog": [{
        "name": "SyslogDataSource",
        "streams": ["Microsoft-Syslog"],
        "facilityNames": ["auth", "authpriv"],
        "logLevels": ["Warning", "Error", "Critical"]
      }]
    },
    "destinations": {
      "logAnalytics": [{
        "workspaceResourceId": "/subscriptions/.../workspaces/sentinel-ws",
        "name": "sentinel-destination"
      }]
    },
    "dataFlows": [{
      "streams": ["Microsoft-SecurityEvent", "Microsoft-Syslog"],
      "destinations": ["sentinel-destination"]
    }]
  }
}

Data Transformation

Transform at Ingestion

DCRs can transform data BEFORE it's stored - filter rows, drop columns, parse fields. This reduces storage costs and improves query performance.

Transformation Query
// In DCR transformKql property:
source
| where EventID in (4624, 4625, 4688)  // Keep only specific events
| extend ParsedAccount = extract(@"^([^\\]+)\\(.+)$", 2, TargetUserName)
| project-away RawEventData  // Drop large unnecessary field

Interview Questions

Legacy (MMA/OMS Agent):

  • Configuration in workspace or agent
  • Limited transformation capability
  • All-or-nothing collection

Modern (AMA + DCR):

  • Centralized configuration via Azure
  • Ingestion-time transformation with KQL
  • Granular filtering (specific Event IDs, facilities)
  • Multi-destination routing
  • Better performance and security
