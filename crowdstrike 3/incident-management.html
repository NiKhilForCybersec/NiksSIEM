Detection vs Incident

CrowdStrike Detection Hierarchy: DETECTION (Single Alert): ├── Single security event ├── One technique/behavior detected ├── Associated with one process/activity └── Example: "PowerShell.exe executed encoded command" INCIDENT (Grouped Detections): ├── Multiple related detections ├── Grouped by endpoint and timeframe ├── Shows attack progression ├── Example: "WIZARD SPIDER campaign on HOST01" HIERARCHY: ┌─────────────────────────────────────────────────────────┐ │ INCIDENT │ │ ┌─────────────────┐ ┌─────────────────┐ │ │ │ Detection 1 │ │ Detection 2 │ ... │ │ │ (Initial Access)│ │ (Execution) │ │ │ └─────────────────┘ └─────────────────┘ │ │ │ │ │ │ ▼ ▼ │ │ ┌─────────────────┐ ┌─────────────────┐ │ │ │ Process Tree │ │ Process Tree │ │ │ │ (Causality) │ │ (Causality) │ │ │ └─────────────────┘ └─────────────────┘ │ └─────────────────────────────────────────────────────────┘

Queue Filtering

Detection Queue Filters: TIME FILTERS: ├── Last 24 hours (default) ├── Last 7 days ├── Last 30 days ├── Custom range └── Real-time (auto-refresh) SEVERITY FILTERS: ├── Critical (immediate action) ├── High (priority review) ├── Medium (standard review) ├── Low (routine review) └── Informational (tracking) STATUS FILTERS: ├── New (unreviewed) ├── In Progress (assigned) ├── Reopened (needs re-review) ├── True Positive (confirmed) ├── False Positive (tuning candidate) └── Ignored (dismissed) ADDITIONAL FILTERS: ├── Host group ├── Detection type ├── MITRE tactic/technique ├── Assigned analyst ├── Sensor action (blocked/detected) └── Scenario (e.g., Ransomware)

Queue Prioritization

Triage Priority Matrix: PRIORITY 1 - CRITICAL (Immediate): ├── Severity: Critical ├── Confidence: High ├── Scenario: Ransomware, Active intrusion ├── Action: Blocked (failed) or Detected └── SLA: 15 minutes PRIORITY 2 - HIGH (Urgent): ├── Severity: High ├── Confidence: Medium-High ├── Scenario: Credential theft, C2 ├── Multiple detections same host └── SLA: 1 hour PRIORITY 3 - MEDIUM (Standard): ├── Severity: Medium ├── Confidence: Medium ├── Scenario: Reconnaissance, Persistence ├── Single detection └── SLA: 4 hours PRIORITY 4 - LOW (Routine): ├── Severity: Low/Informational ├── Confidence: Low-Medium ├── Scenario: Policy violation ├── Known behavior under review └── SLA: 24 hours

Detection Investigation

Investigation Workflow

Detection Investigation Steps: 1. INITIAL TRIAGE (2-5 minutes) ├── Read detection summary ├── Review severity and confidence ├── Check MITRE ATT&CK mapping ├── Note affected host and user └── Quick assessment: legitimate or suspicious? 2. CONTEXT GATHERING (5-10 minutes) ├── Review process tree (causality chain) ├── Examine command line arguments ├── Check parent process legitimacy ├── Review file metadata and hashes ├── Check for related detections └── Query Falcon Intelligence for IOCs 3. SCOPING (10-15 minutes) ├── Search for same activity on other hosts ├── Check user activity timeline ├── Review network connections ├── Look for lateral movement indicators └── Assess data access/exfiltration risk 4. DETERMINATION ├── True Positive → Escalate/Respond ├── False Positive → Document and tune ├── Needs More Info → Dig deeper └── Suspicious → Elevate for review 5. DOCUMENTATION ├── Update detection status ├── Add investigation notes ├── Tag with relevant labels ├── Assign for follow-up if needed └── Create ticket in ITSM if required

Process Tree Analysis

Process Tree (Causality Chain): The process tree shows parent-child relationships: Example - Malicious Macro: ┌────────────────────────────────────────────────────────┐ │ OUTLOOK.EXE (Parent) │ │ │ │ │ └─▶ WINWORD.EXE (Opened attachment) │ │ │ │ │ └─▶ POWERSHELL.EXE (Macro execution) │ │ │ │ │ └─▶ CMD.EXE (System discovery) │ │ │ │ │ ├─▶ WHOAMI.EXE │ │ ├─▶ IPCONFIG.EXE │ │ └─▶ NET.EXE (user enumeration) │ └────────────────────────────────────────────────────────┘ KEY INDICATORS IN PROCESS TREE: ├── Unusual parent-child relationships ├── Suspicious command line arguments ├── Encoded/obfuscated commands ├── Network connections from process ├── File writes (dropped files) ├── Registry modifications └── Injection into other processes EXPANSION OPTIONS: ├── View full command line ├── See network activity ├── View file operations ├── Check DNS queries ├── See registry changes └── View module loads

Scoping and Pivoting

Searching for Related Activity

Pivoting from Detection: 1. SAME HOST ACTIVITY Investigate → Event Search └── Query: aid:[agent_id] | Timeline view 2. SAME USER ACROSS HOSTS Investigate → Event Search └── Query: UserName:"DOMAIN\\jsmith" └── Check: Other compromised systems? 3. SAME IOC ACROSS ENVIRONMENT Investigate → Event Search └── Query: SHA256:"abc123..." OR DomainName:"malicious.com" └── Check: Scope of compromise? 4. SAME TECHNIQUE Investigate → Event Search └── Query: event_platform:win CommandLine:"*-enc*" └── Check: Other affected systems? 5. LATERAL MOVEMENT INDICATORS Check for connections to: ├── Domain Controllers (ports 88, 389, 445) ├── File servers (port 445) ├── Other workstations (ports 135, 445) └── Remote management (ports 5985, 5986, 3389)

Containment Decision Tree

Containment Decision Framework: START: Active threat detected? │ ├─▶ YES → High severity + High confidence? │ │ │ ├─▶ YES → Contain immediately │ │ └── Document reason, notify user │ │ │ └─▶ NO → Active data exfiltration? │ │ │ ├─▶ YES → Contain immediately │ │ │ └─▶ NO → Ransomware indicators? │ │ │ ├─▶ YES → Contain immediately │ │ │ └─▶ NO → Investigate before containing │ └─▶ NO → Historical detection? │ └─▶ Investigate, scope, then decide CONTAINMENT CHECKLIST: □ Document justification □ Notify affected user/manager □ Create incident ticket □ Preserve evidence before remediation □ Plan for business continuity □ Set reminder to lift containment

Interview Questions

  1. Initial Assessment (2-5 min): Review detection summary, severity, confidence, MITRE mapping
  2. Context Gathering (5-10 min): Examine process tree, command lines, parent process, related detections
  3. Scoping (10-15 min): Search for same activity across environment, check lateral movement
  4. Determination: True Positive (respond) or False Positive (document, tune)
  5. Documentation: Update status, add notes, create ticket if needed

Key indicators in process tree analysis:

  • Unusual parent-child: Office app spawning PowerShell, cmd from services
  • Command line arguments: Encoded commands, download cradles, suspicious flags
  • Process chain depth: Multiple levels of spawned processes
  • Network activity: External connections from unexpected processes
  • File operations: Dropped files, especially in temp folders

Contain immediately when:

  • High severity + high confidence active threat
  • Ransomware indicators (encryption activity)
  • Active data exfiltration
  • C2 beaconing with data transfer
  • Lateral movement in progress

Always: Document justification, notify user, preserve evidence, plan for business continuity.

Scoping via Event Search:

  1. Same host: Query by agent ID for full timeline
  2. Same user: Search username across all hosts
  3. Same IOC: Search hash, domain, IP across environment
  4. Same technique: Search for similar command patterns

Also check for lateral movement connections to DCs, file servers, other workstations.
