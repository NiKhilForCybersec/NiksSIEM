CrowdStrike Falcon Fundamentals

Event Search Syntax

Basic Search: event_simpleName=ProcessRollup2 FileName=powershell.exe Field Operators: = exact match != not equal =* wildcard match =~ case insensitive >< range Logical Operators: AND - both conditions OR - either condition NOT - exclude Examples: // Find PowerShell with encoded commands event_simpleName=ProcessRollup2 FileName=powershell.exe CommandLine=*-enc* // Find network connections to IP event_simpleName=NetworkConnectIP4 RemoteIP=1.2.3.4 // Find file writes to temp event_simpleName=FileWritten FilePath=*\\Temp\\* // Find process on specific host event_simpleName=ProcessRollup2 ComputerName=WORKSTATION01

Response Actions

Network Containment

Network Contain: - Blocks all network traffic except CrowdStrike - Device can still send telemetry - Prevents lateral movement - Can run RTR commands Contain Methods: 1. Interview Questions

  • IOC (Indicator of Compromise): Static indicators like hashes, IPs, domains - evidence of past compromise
  • IOA (Indicator of Attack): Behavioral patterns that detect attack techniques - stops attacks in progress

Key point: IOAs can detect unknown/new attacks because they focus on behavior, not signatures.

  • Instant updates: New protections deployed immediately
  • Global visibility: See threats across all customers
  • Lightweight agent: Minimal endpoint impact
  • Scalable analysis: Cloud ML and analytics
  • No maintenance: No signature updates to manage

Exclusions Management

Configuration → Exclusions

Exclusion Types

ML Exclusions: - Exclude files from ML analysis - Path-based or hash-based - Use for known-good software IOA Exclusions: - Exclude specific behaviors - Rule-specific exclusions - Process + behavior combinations Sensor Visibility Exclusions: - Exclude from data collection - Use sparingly (reduces visibility) - For high-volume benign activity

Best Practices

  • Be specific: Narrow exclusions reduce risk
  • Hash over path: Hash exclusions are more specific
  • Document everything: Track why exclusions exist
  • Regular review: Audit exclusions quarterly
  • Test first: Verify exclusion works as expected

Common Issues

Sensor Issues

Sensor Not Connecting: 1. Check sensor service - Windows: sc query csagent - Linux: systemctl status falcon-sensor 2. Verify network connectivity - Cloud connectivity check - Proxy configuration - Firewall rules 3. Check sensor logs - Windows: Event Viewer → CrowdStrike - Linux: /var/log/falcon/ 4. Reinstall if needed - Uninstall completely - Fresh installation

False Positive Handling

FP Response Process: 1. Verify it's truly benign - Investigate thoroughly - Check with application owner - Validate expected behavior 2. Choose exclusion type - Hash (most specific) - Path (if hash changes) - Process + path 3. Create exclusion - Document reason - Link to ticket - Set scope (global vs group) 4. Monitor - Verify exclusion works - Watch for related activity
  • All network traffic blocked except CrowdStrike cloud
  • Host can still send telemetry and receive commands
  • RTR sessions still work for investigation
  • Prevents lateral movement and C2 communication
  • User sees no network connectivity
  • Must manually lift containment when ready
  • Behavioral detection: Detects encryption patterns
  • Process blocking: Stops suspicious processes
  • Volume shadow protection: Prevents VSS deletion
  • Known ransomware signatures: Hash-based blocking
  • MFA bombing detection: Stops initial access
