Integration Examples

# Python - Query indicators import requests BASE_URL = "https://api.crowdstrike.com" TOKEN = "your_oauth_token" headers = { "Authorization": f"Bearer {TOKEN}", "Content-Type": "application/json" } # Search for domain indicators def search_indicators(domain): params = { "filter": f"type:'domain'+value:'{domain}'", "limit": 100 } response = requests.get( f"{BASE_URL}/intel/combined/indicators/v1", headers=headers, params=params ) return response.json() # Get actor details def get_actor(actor_name): # First search for actor ID search_response = requests.get( f"{BASE_URL}/intel/queries/actors/v1", headers=headers, params={"q": f"name:'{actor_name}'"} ) actor_ids = search_response.json()["resources"] # Get full details details_response = requests.get( f"{BASE_URL}/intel/entities/actors/v1", headers=headers, params={"ids": actor_ids} ) return details_response.json() # SIEM Integration - Export IOCs for Splunk def export_iocs_for_siem(actor_name): # Get actor indicators actor = get_actor(actor_name) indicators = actor.get("resources", [{}])[0].get("indicators", []) # Format for Splunk lookup csv_lines = ["type,value,actor,confidence"] for ioc in indicators: csv_lines.append( f"{ioc['type']},{ioc['value']},{actor_name},{ioc['confidence']}" ) return "\n".join(csv_lines)

Detection Rule Creation

From Intelligence to Detection: EXAMPLE: WIZARD SPIDER Campaign 1. INTELLIGENCE REPORT SAYS: "WIZARD SPIDER uses encoded PowerShell to download Cobalt Strike beacon from compromised WordPress sites" 2. EXTRACT TTPs: ├── T1059.001 - PowerShell ├── T1027 - Obfuscated Files ├── T1071.001 - Web Protocols └── T1189 - Drive-by Compromise 3. CREATE CUSTOM IOA: Name: WIZARD_SPIDER_PowerShell_Download Platform: Windows Condition: ProcessName = "powershell.exe" AND CommandLine contains ("downloadstring" OR "IEX" OR "Invoke-Expression") AND CommandLine contains ("/wp-content/" OR "/wp-includes/") Action: Detect and Alert 4. ADD CUSTOM IOCs: ├── Known C2 domains ├── Cobalt Strike beacon hashes └── Infrastructure IP ranges 5. CREATE HUNT QUERY: // Search for related activity event_platform: win | filter process_name = "powershell.exe" | filter command_line matches "(?i)(downloadstring|invoke-webrequest).*wp-(content|includes)" | group by aid, command_line 6. MEASURE EFFECTIVENESS: ├── True positive rate ├── False positive rate ├── Coverage against campaign TTPs └── Time to detect

Interview Questions

CrowdStrike uses [ADJECTIVE] [ANIMAL] format:

  • BEAR: Russia (FANCY BEAR, COZY BEAR)
  • PANDA: China (WICKED PANDA)
  • KITTEN: Iran (CHARMING KITTEN)
  • CHOLLIMA: North Korea
  • SPIDER: eCrime/Ransomware (WIZARD SPIDER)

The animal indicates nation-state/motivation, adjective is unique identifier.

  1. Search IOCs: Query hashes, domains, IPs found in incident
  2. Identify actor: Check for attribution, known campaigns
  3. Understand TTPs: Review actor profile for expected techniques
  4. Find related IOCs: Pivot to discover additional indicators
  5. Submit samples: Analyze unknown files in sandbox
  6. Hunt: Use TTPs to search for additional compromise
  • IOCs (Indicators of Compromise): Atomic, specific - hashes, IPs, domains. Easy to change, short lifespan.
  • TTPs (Tactics, Techniques, Procedures): Behavioral patterns - how attackers operate. Harder to change, longer detection value.

TTPs are more valuable for detection because attackers can easily change IOCs but their behavior patterns are harder to modify.

  1. Collect: Gather intel from multiple sources
  2. Process: Normalize, deduplicate, validate relevance
  3. Analyze: Correlate with environment, identify gaps
  4. Disseminate: Push IOCs to tools, create detections, brief stakeholders
  5. Feedback: Measure effectiveness, retire stale intel

Key: Transform intelligence into actionable defenses (detections, hunts, IOC blocks).

Static Analysis: PE headers, strings, imports, packer detection, YARA matches

Dynamic Analysis:

  • Process execution and injection
  • Network connections and DNS queries
  • File system and registry changes
  • API calls and system interactions

Intelligence: Malware family, actor attribution, MITRE ATT&CK mapping, extracted IOCs

  1. Read report: Understand adversary, targets, TTPs
  2. Extract TTPs: Map to MITRE ATT&CK techniques
  3. Create Custom IOA: Build rule for behavioral detection
  4. Add IOCs: Upload known hashes, domains, IPs
  5. Build hunt query: Search for historical activity
  6. Measure: Track true/false positives, tune as needed
