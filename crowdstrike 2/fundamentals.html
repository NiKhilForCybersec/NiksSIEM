CrowdStrike Falcon Fundamentals

CrowdStrike Falcon is a cloud-native endpoint protection platform that combines next-gen antivirus, EDR, and threat hunting capabilities. Master the fundamentals for effective security operations.

What is CrowdStrike Falcon?

CrowdStrike Falcon provides:

  • Falcon Prevent: Next-gen antivirus (NGAV)
  • Falcon Insight: Endpoint detection and response (EDR)

CrowdStrike Advantages: Cloud-Native Architecture: - Single lightweight agent - Cloud-delivered updates - Instant protection updates - No signature updates needed AI/ML-Powered Detection: - Behavioral analytics - Indicator of Attack (IOA) - Machine learning models - No reliance on signatures Threat Intelligence: - CrowdStrike Intelligence - OverWatch threat hunting - Adversary profiles - Real-time threat feeds

Process Tree Analysis

Example Process Tree: explorer.exe (User Shell) │ └── outlook.exe (Email Client) │ └── [!] WINWORD.EXE (Document) │ └── [!] cmd.exe (Shell) │ └── [!] powershell.exe ← DETECTION │ ├── Network: malicious.com └── File: C:\...\payload.exe [!] = Part of detection chain Investigation Questions: 1. What started this chain? 2. Is the parent process legitimate? 3. What did the process do? 4. Is there network C2? 5. Were files dropped?

Event Search Syntax

Basic Search: event_simpleName=ProcessRollup2 FileName=powershell.exe Field Operators: = exact match != not equal =* wildcard match =~ case insensitive >< range Logical Operators: AND - both conditions OR - either condition NOT - exclude Examples: // Find PowerShell with encoded commands event_simpleName=ProcessRollup2 FileName=powershell.exe CommandLine=*-enc* // Find network connections to IP event_simpleName=NetworkConnectIP4 RemoteIP=1.2.3.4 // Find file writes to temp event_simpleName=FileWritten FilePath=*\\Temp\\* // Find process on specific host event_simpleName=ProcessRollup2 ComputerName=WORKSTATION01

Response Actions

Network Containment

Network Contain: - Blocks all network traffic except CrowdStrike - Device can still send telemetry - Prevents lateral movement - Can run RTR commands Contain Methods: 1. From Detection: "Network Contain" button 2. From Host Details: "Network Contain Host" 3. Via RTR: containment commands 4. Via API: Automated containment Lifting Containment: - Verify threat is remediated - "Lift Containment" from host details - Monitor for suspicious activity after

Real Time Response (RTR)

RTR Session: Investigate → Hosts → [Host] → RTR Basic Commands (Read-only): - cat - View file contents - cd - Change directory - dir/ls - List files - filehash - Get file hash - ipconfig/ifconfig - Network config - netstat - Network connections - ps - Process list - reg query - Registry query Active Response Commands: - kill - Terminate process - rm/del - Delete file - reg delete - Delete registry key - remediate - Remove threat - restart - Restart host Admin Commands: - get - Download file - put - Upload file - run - Execute script - xmemdump - Memory dump

Falcon OverWatch

OverWatch is CrowdStrike's managed threat hunting service.

OverWatch Capabilities

  • 24/7 Hunting: Continuous human-led threat hunting
  • Novel Threat Detection: Find threats that automation misses
  • Expert Analysis: CrowdStrike threat hunters
  • Rapid Notification: Alert on emerging threats

OverWatch Alerts

OverWatch Detection Types: Intrusion Activity: - Active adversary in environment - Immediate escalation - Highest priority Suspicious Activity: - Potentially malicious behavior - Requires investigation - May be benign State-Sponsored Activity: - Nation-state adversary indicators - Advanced persistent threat - Strategic targeting

Interview Questions

  • IOC (Indicator of Compromise): Static indicators like hashes, IPs, domains - evidence of past compromise
  • IOA (Indicator of Attack): Behavioral patterns that detect attack techniques - stops attacks in progress

Key point: IOAs can detect unknown/new attacks because they focus on behavior, not signatures.

  • Instant updates: New protections deployed immediately
  • Global visibility: See threats across all customers
  • Lightweight agent: Minimal endpoint impact
  • Scalable analysis: Cloud ML and analytics
  • No maintenance: No signature updates to manage
  1. Review detection: Severity, type, affected host
  2. Examine process tree: How did it start? Parents?
  3. Check command line: Suspicious arguments?
  4. Look at network: External connections?
  5. Check for persistence: Registry, scheduled tasks?
  6. Scope assessment: Other hosts affected?
  7. Take action: Contain, kill, quarantine
