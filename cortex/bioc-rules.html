BIOC Rule Structure

Rule Components

Every BIOC rule consists of the following elements:

# BIOC Rule JSON Structure
{
    "name": "Rule Name",
    "description": "Detailed description of what the rule detects",
    "severity": "high",  // critical, high, medium, low, informational
    "category": "malware",  // malware, exploitation, persistence, etc.
    "mitre_tactic": "TA0003",
    "mitre_technique": "T1059.001",
    "xql_query": "dataset = xdr_data | filter ...",
    "enabled": true,
    "alert_config": {
        "group_by": ["agent_hostname"],
        "time_window": "5m",
        "threshold": 1
    }
}

Severity Levels

Severity Description Response Time Example
Critical Confirmed active threat Immediate Ransomware execution, C2 communication
High Likely malicious activity 15 minutes Credential dumping, lateral movement
Medium Suspicious behavior 1 hour Unusual PowerShell, persistence mechanism
Low Anomalous activity 4 hours Policy violation, minor deviation
Informational For tracking/audit 24 hours Administrative action, compliance log

BIOC Rule Examples

Credential Access Detection

# BIOC: LSASS Memory Access Detection
# Detects processes accessing LSASS memory (credential dumping indicator)
dataset = xdr_data 
| filter event_type = ENUM.EVENT_TYPE_OPEN_REMOTE_PROCESS_HANDLE
| filter target_process_name contains "lsass"
| filter action_process_image_name not in (
    "MsMpEng.exe",
    "csrss.exe",
    "svchost.exe",
    "lsass.exe",
    "wininit.exe"
)
| filter action_process_requested_access contains "PROCESS_VM_READ"
| fields agent_hostname, 
         action_process_image_name,
         action_process_image_path,
         action_process_command_line,
         target_process_name,
         action_process_username,
         action_process_sha256
| limit 100
# Rule Configuration
{
    "name": "LSASS Memory Access - Possible Credential Dumping",
    "severity": "high",
    "mitre_tactic": "TA0006",
    "mitre_technique": "T1003.001"
}

Execution - Encoded PowerShell

# BIOC: Encoded PowerShell Command Detection
# Detects base64 encoded PowerShell commands often used by attackers
dataset = xdr_data 
| filter event_type = ENUM.EVENT_TYPE_PROCESS
| filter action_process_image_name in ("powershell.exe", "pwsh.exe")
| filter action_process_command_line contains "-enc" or
         action_process_command_line contains "-EncodedCommand" or
         action_process_command_line contains "FromBase64String"
| filter action_process_command_line length > 500
| fields agent_hostname,
         action_process_image_path,
         action_process_command_line,
         actor_process_image_name,
         actor_process_command_line,
         action_process_username
| limit 100
# Rule Configuration
{
    "name": "Encoded PowerShell Execution",
    "severity": "medium",
    "mitre_tactic": "TA0002",
    "mitre_technique": "T1059.001"
}

Persistence - Registry Run Keys

# BIOC: Suspicious Registry Run Key Modification
# Detects additions to registry run keys for persistence
dataset = xdr_data 
| filter event_type = ENUM.EVENT_TYPE_REGISTRY_SET_VALUE
| filter action_registry_key_name contains "\\CurrentVersion\\Run" or
         action_registry_key_name contains "\\CurrentVersion\\RunOnce"
| filter action_process_image_name not in (
    "msiexec.exe",
    "setup.exe",
    "TrustedInstaller.exe"
)
| fields agent_hostname,
         action_registry_key_name,
         action_registry_value_name,
         action_registry_data,
         action_process_image_name,
         action_process_image_path,
         action_process_username
| limit 100
# Rule Configuration
{
    "name": "Registry Run Key Persistence",
    "severity": "medium",
    "mitre_tactic": "TA0003",
    "mitre_technique": "T1547.001"
}

Lateral Movement - PsExec Activity

# BIOC: PsExec Remote Execution Detection
# Detects PsExec or similar remote execution tools
dataset = xdr_data 
| filter event_type = ENUM.EVENT_TYPE_PROCESS
| filter (action_process_image_name = "psexec.exe" or
          action_process_image_name = "psexec64.exe" or
          action_process_image_name = "paexec.exe") or
         (action_process_image_name = "PSEXESVC.exe")
| fields agent_hostname,
         action_process_image_name,
         action_process_command_line,
         actor_process_image_name,
         action_process_username,
         action_remote_ip,
         action_process_sha256
| limit 100
# Rule Configuration
{
    "name": "PsExec Remote Execution Detected",
    "severity": "high",
    "mitre_tactic": "TA0008",
    "mitre_technique": "T1570"
}

Defense Evasion - Security Tool Tampering

# BIOC: Security Tool Termination Attempt
# Detects attempts to stop security services or kill security processes
dataset = xdr_data 
| filter event_type = ENUM.EVENT_TYPE_PROCESS
| filter action_process_command_line contains "stop" or
         action_process_command_line contains "disable" or
         action_process_command_line contains "taskkill"
| filter action_process_command_line contains "defender" or
         action_process_command_line contains "sophos" or
         action_process_command_line contains "symantec" or
         action_process_command_line contains "crowdstrike" or
         action_process_command_line contains "carbon" or
         action_process_command_line contains "sentinel" or
         action_process_command_line contains "mcafee" or
         action_process_command_line contains "cortex" or
         action_process_command_line contains "traps"
| fields agent_hostname,
         action_process_image_name,
         action_process_command_line,
         actor_process_image_name,
         action_process_username
| limit 100
# Rule Configuration
{
    "name": "Security Tool Tampering Attempt",
    "severity": "critical",
    "mitre_tactic": "TA0005",
    "mitre_technique": "T1562.001"
}

Exfiltration - Large Data Transfer

# BIOC: Large Outbound Data Transfer
# Detects unusually large data transfers to external IPs
dataset = xdr_data 
| filter event_type = ENUM.EVENT_TYPE_NETWORK
| filter action_external_direction = "OUTBOUND"
| filter action_remote_ip != "10.*" and 
         action_remote_ip != "172.16.*" and 
         action_remote_ip != "192.168.*"
| filter action_total_bytes_sent > 104857600  // 100 MB
| fields agent_hostname,
         action_process_image_name,
         action_remote_ip,
         action_remote_port,
         action_total_bytes_sent,
         action_process_username
| sort desc action_total_bytes_sent
| limit 50
# Rule Configuration
{
    "name": "Large Outbound Data Transfer",
    "severity": "high",
    "mitre_tactic": "TA0010",
    "mitre_technique": "T1048"
}

Command and Control - Suspicious DNS

# BIOC: DNS Tunneling Detection
# Detects unusually long DNS queries that may indicate DNS tunneling
dataset = xdr_data 
| filter event_type = ENUM.EVENT_TYPE_DNS
| filter action_query_type = "TXT" or 
         action_dns_query_name length > 50
| filter action_dns_query_name not contains "microsoft.com" and
         action_dns_query_name not contains "windows.com" and
         action_dns_query_name not contains "office.com"
| alter query_length = length(action_dns_query_name)
| filter query_length > 60
| fields agent_hostname,
         action_dns_query_name,
         query_length,
         action_process_image_name,
         action_remote_ip
| sort desc query_length
| limit 100
# Rule Configuration
{
    "name": "Possible DNS Tunneling",
    "severity": "high",
    "mitre_tactic": "TA0011",
    "mitre_technique": "T1071.004"
}

Best Practices

Rule Development Workflow

1

Research

Study the threat behavior, MITRE technique, and available telemetry

2

Query Development

Write XQL query in Investigation module, test against historical data

3

Validate

Review results for false positives, refine filters and exclusions

4

Deploy

Create BIOC rule with appropriate severity and alerting config

5

Monitor

Track alert volume, false positive rate, and detection efficacy

6

Tune

Add exclusions, adjust thresholds, update based on feedback

Common Exclusion Patterns

# Exclude known legitimate tools
| filter action_process_image_name not in (
    "MsMpEng.exe",      # Windows Defender
    "CylanceSvc.exe",   # Cylance
    "CrowdStrike*",     # CrowdStrike
    "SentinelOne*"      # SentinelOne
)
# Exclude trusted paths
| filter action_process_image_path not contains "\\Windows\\System32\\"
| filter action_process_image_path not contains "\\Program Files\\Microsoft\\"
# Exclude service accounts
| filter action_process_username not in (
    "NT AUTHORITY\\SYSTEM",
    "NT AUTHORITY\\LOCAL SERVICE",
    "NT AUTHORITY\\NETWORK SERVICE"
)
# Exclude by hash (verified safe)
| filter action_process_sha256 not in (
    "abc123...",  # Known good hash
    "def456..."   # Known good hash
)
# Exclude signed binaries (use with caution)
| filter action_process_signature_status != "Signed"

Q: How do you create and tune custom detection rules in Cortex XDR?

A: I follow a structured approach. First, I research the threat behavior I want to detect and identify which MITRE ATT&CK technique it maps to. Then I use the Investigation module to write and test XQL queries against historical data to ensure the logic captures the behavior. I analyze the results for false positives and add appropriate exclusions for legitimate tools and processes. Once validated, I create a BIOC rule with the appropriate severity level and configure alerting thresholds. After deployment, I monitor the alert volume and false positive rate, continuously tuning the rule based on analyst feedback. I also maintain documentation linking each rule to its MITRE technique for coverage analysis.

Q: What's the difference between IOCs and BIOCs in Cortex XDR?

A: IOCs (Indicators of Compromise) are static indicators like file hashes, IP addresses, or domain names that represent known threats. They're reactive, meaning you need to already know about a threat to detect it. BIOCs (Behavioral Indicators of Compromise) detect suspicious behaviors regardless of the specific malware or tool used. For example, instead of blocking a known Mimikatz hash (IOC), a BIOC rule detects any process accessing LSASS memory for reading, which would catch both Mimikatz and any new credential dumping tool. BIOCs are more resilient against attackers who modify their tools to evade hash-based detection.
