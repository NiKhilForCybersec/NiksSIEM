Data Collection

Configure and manage data collection in Cortex XDR including agent data, third-party sources, and cloud integrations.

Data Sources

Supported Data Sources

  • Cortex XDR Agent: Endpoint telemetry, process events, file events, network connections
  • Cloud Logs: AWS CloudTrail, Azure Activity, GCP Audit
  • Network Devices: Firewall logs, proxy logs, DNS logs
  • Identity Sources: Active Directory, Azure AD, Okta
  • Third-Party EDR: Integration with other endpoint solutions

Collection Methods

Agent-Based Collection

The Cortex XDR agent collects comprehensive endpoint data including:

  • Process creation and termination events
  • File system modifications
  • Registry changes (Windows)
  • Network connections and DNS queries
  • User authentication events

Agentless Collection

For cloud and network data:

  • API-based ingestion from cloud providers
  • Syslog forwarding from network devices
  • Broker VM for on-premises log collection

Configuration

Console Path

Settings → Configurations → Data Collection

Key Settings

  • Collection Profiles: Define what data to collect per endpoint group
  • Data Filters: Exclude noisy or sensitive data
  • Retention: Configure how long raw data is stored
